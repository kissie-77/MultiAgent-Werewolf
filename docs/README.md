# 文档导航

`docs/` 现在按用途分区，避免所有 Markdown 平铺在根目录。

## 目录规则

- `architecture/`：工程结构、模块边界、Prompt 结构、重构方案
- `memory/`：记忆板块开发记录、记忆相关设计与阶段笔记
- `evaluation/`：评测设计、验证结果、评测模块方案
- `reports/`：专项分析报告、性能报告、问题修复报告、使用说明
- `archive/`：历史文档、个人过程稿、旧方案、已归档资料

## 放置原则

- 新文档如果描述“代码应该怎么组织”，放 `architecture/`
- 新文档如果描述“memory 怎么设计/怎么演进”，放 `memory/`
- 新文档如果描述“怎么评测/评测结果是什么”，放 `evaluation/`
- 新文档如果是专题结论、排查报告、性能分析、使用说明，放 `reports/`
- 已经过时、仅保留参考价值、或不再作为当前执行依据的文档，放 `archive/`
