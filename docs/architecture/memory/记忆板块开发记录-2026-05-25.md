## 今日目标

在不引入外部记忆依赖的前提下，为项目搭建可继续演进的记忆管理框架，并保证其符合既定六大板块架构约束。

## 今日完成内容

### 1. 完成四层记忆框架

已实现并接入以下四层记忆：

- 工作记忆 `WorkingMemory`
- 情景记忆 `EpisodicMemory`
- 语义记忆 `SemanticMemory`
- 程序记忆 `ProceduralMemory`

并由 `MemoryManager` 统一调度。

### 2. 完成生命周期闭环

已接通以下生命周期钩子：

- `on_game_start()`：开局注入跨局经验与程序记忆
- `on_round_end()`：回合结束压缩工作记忆
- `on_game_end()`：局结束更新语义记忆并可触发候选提炼

### 3. 完成运行期记忆输入采集

当前已支持以下信息进入记忆流：

- Agent 决策记录
- 白天公开发言
- 玩家最近可见关键事件

### 4. 完成情景到语义的规则式提炼管线

`extract_semantic_candidates()` 已从占位逻辑升级为规则式提炼逻辑，可生成：

- 关键局势复盘
- 决策经验
- 胜利经验
- 失败反思

### 5. 完成语义卡片管理增强

已支持：

- 策略卡片 JSON 持久化
- 候选去重
- 相似卡片归并
- 卡片内容合并
- 局后统计更新
    - `weight`
    - `use_count`
    - `win_count`

### 6. 完成程序记忆薄封装

程序记忆已对现有 Prompt 主线做薄封装，支持：

- 获取角色固定规则
- 获取指定计划文本
- 列出所有计划模板
- 生成程序记忆摘要

并已与真实 `plan_name` 对齐，不再固定读取 `default`。

### 7. 完成配置层接入

新增统一记忆配置，支持控制：

- 是否启用记忆框架
- working / episodic / semantic 是否开启
- 工作记忆窗口大小
- 每局注入语义卡片数量
- 语义存储目录
- 是否启用局末提炼

### 8. 完成架构归位

最初实现曾被放在独立 `memory` 模块中，后已按既定六大板块架构要求完成归位：

- 记忆实现现统一位于 `src/llm_werewolf/agent_team/memory/`
- 不再作为独立第七主模块存在
