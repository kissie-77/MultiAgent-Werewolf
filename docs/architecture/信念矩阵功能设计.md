## 1. 背景与目标

### 1.1 要解决的问题

当前 Agent 决策链已具备：

- 结构化发言（`SpeechDecision`，含 `private_thought`）
- 投票意向采集（`VoteIntentionDecision` + `VoteIntentionTracker`）
- 信息隔离（`PlayerObservation` + `EventVisibility`）
- PostGame 说服 / swing 分析（`camp_persuasion`、`vote_swing`）

**缺失**：显式、可持久化、可评测的 **Theory of Mind（心智模型）**——Agent 对「他人身份」与「他人对己认知」的结构化维护，以及 **狼队阵营内战术共识**。

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **B1 一阶信念** | 每个 Agent 维护「我认为 j 是什么身份」 |
| **B2 二阶信念** | 每个 Agent 维护「我认为 i 认为我是什么 / 是否暴露」 |
| **W 狼人私有战术雷达** | 仅狼人可见、仅本人维护：神职定位（W-G）+ 本人暴露雷达（W-E） |
| **与投票意向联合输出** | 同一次 LLM 结构化调用，减少 O(N²) 额外开销 |
| **信息隔离** | 私有行 + 公开事件驱动的列塌缩；狼人 W 面板按座位隔离，狼间不共享 |
| **可复盘可评测** | 引擎侧汇总快照；赛后校准指标 |

### 1.3 非目标（本期不做）

- MultiMind 完整 **MCTS 策略搜索**（需局面模拟器，单独立项）
- 用信念矩阵 **替代** 现有策略 Prompt / Skill 库
- DemoAgent **消费** 信念矩阵做复杂推理（仅需产出合法占位结构供 offline 跑通）

## 2. 概念总览

### 2.1 三类心智状态

```text
┌─────────────────────────────────────────────────────────────────┐
│  每个 Agent（含好人 / 狼人）                                       │
│  ┌─────────────────────┐  ┌─────────────────────┐               │
│  │ B1 一阶（私有）      │  │ B2 二阶（私有）      │               │
│  │ 我认为 j 是什么身份  │  │ 我认为 i 怀疑我什么  │               │
│  └─────────────────────┘  └─────────────────────┘               │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  每个狼人 Agent（私有，与其他狼人隔离）                            │
│  ┌─────────────────────┐  ┌─────────────────────┐               │
│  │ W-G 神职定位         │  │ W-E 暴露雷达（本人） │               │
│  │ 我认为 j 像什么神职  │  │ 我认为 i 对「我」的怀疑 │            │
│  └─────────────────────┘  └─────────────────────┘               │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  引擎 / 复盘（上帝视角，Agent 不可见）                             │
│  beliefs_log：每轮 × 每个 observer 的快照 → 完整 N×N 热力图        │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 是否「全局共享」？

| 说法 | 是否正确 |
|------|----------|
| 运行时全体共改一张矩阵 | ❌ |
| 逻辑上可汇总为 N×N 上帝视图 | ✅（仅日志 / 评测） |
| 每个 Agent 只维护自己的 B1/B2 行 | ✅ |
| 每个狼人只维护自己的 W-G / W-E（狼间不 merge） | ✅ |
| 公开翻牌后所有人对某列信念趋同 | ✅ |

## 3. B1：一阶信念矩阵

### 3.1 语义

**问题**：在我（observer）目前掌握的信息下，玩家 j 各是什么身份？

| 维度 | 含义 |
|------|------|
| **行 i** | 信念持有者 = 座位 i 的 Agent（仅 i 可见该行） |
| **列 j** | 被判断对象 = 座位 j |
| **单元格 B1[i][j]** | i 对 j 的身份概率分布（首期可简化为 `wolf_probability`） |

### 3.2 单元格内容

**完整版（目标形态）**：

```json
{
  "target_seat": 3,
  "role_distribution": {
    "Werewolf": 0.45,
    "Seer": 0.08,
    "Witch": 0.12,
    "Guard": 0.05,
    "Hunter": 0.00,
    "Villager": 0.30
  },
  "confidence": 0.72,
  "note": "发言前后矛盾，投票跟 4 号"
}
```

**首期 MVP（与现有 `BeliefEntry` 对齐）**：

```json
{
  "target_seat": 3,
  "wolf_probability": 0.45,
  "note": "发言前后矛盾"
}
```

非狼身份首期用 `1 - wolf_probability` 粗表示「偏好人」，二期再展开 `role_distribution`。

### 3.3 对角线与死亡列

| 情况 | B1[i][i] | B1[*][j]（j 已死且身份公开） |
|------|----------|------------------------------|
| 已知自己身份 | 对应 role = 1.0 | — |
| j 被投/杀且翻牌 | — | 所有存活者该列 one-hot 为真实身份 |

### 3.4 6 人局示例（上帝视角摘要）

存活：1 女巫、2 预言家、3/4 双狼、5/6 村民；6 号已出局（村民）。

```text
         │  1号    2号    3号    4号    5号    6号(已知村民)
─────────┼──────────────────────────────────────────────────
1号 女巫  │  自     低狼    高狼    高狼    低狼    已知
2号 预言家│  低狼    自     低狼*   高狼    低狼    已知
3号 狼    │  偏神    想抗推   队友    队友    可推    已知
4号 狼    │  偏神    想抗推   队友    自     可推    已知
5号 村民  │  略疑    略疑    可疑    更疑    自     已知

* 若 2 号验过 3 号为好人，则仅 2 号行上 3 列狼概率 ≈ 0
```

### 3.5 运行时存储

```text
Agent.belief_state.first_order: dict[int, BeliefEntry]   # key = target_seat
GameState.belief_log: list[BeliefSnapshot]                 # 上帝视角，Agent 不可读
```

每个 Agent **只存一行**（对所有 j 的列表），不存整张 N×N。

## 4. B2：二阶信念矩阵

### 4.1 语义

**问题**：我认为 **observer i** 认为 **我** 是什么身份 / 是否已怀疑我是狼？

| 维度 | 含义 |
|------|------|
| **行** | 固定为「我自己」（当前 Agent） |
| **列 i** | 我眼中的 observer（其他存活玩家） |
| **单元格 B2[me][i]** | 我认为 i 怀疑我是狼的概率（首期） |

### 4.2 单元格内容

**首期 MVP**：

```json
{
  "observer_seat": 2,
  "suspects_me_as_wolf": 0.65,
  "note": "2 号发言点名攻击我"
}
```

**完整版（二期）**：

```json
{
  "observer_seat": 2,
  "believes_me_role": {
    "Werewolf": 0.65,
    "Seer": 0.05,
    "Villager": 0.30
  },
  "exposure_risk": 0.65
}
```

### 4.3 用途

| 场景 | 如何使用 B2 |
|------|-------------|
| 发言策略 | `exposure_risk` 高 → 防守 / 转移 / 倒钩 |
| 投票布局 | 预测谁会把票挂到自己 |
| 狼队协调 | 仅能通过公开/狼队频道发言协调；W 面板不在狼间同步 |

### 4.4 运行时存储

```text
Agent.belief_state.second_order: dict[int, ExposureEntry]  # key = observer_seat
```

## 5. W：狼人私有战术雷达

仅 **存活狼人** 在 **私密决策上下文**（mind state / 夜间讨论 prompt）读写**本人**面板；其他狼人不可见、不可写入。

### 5.1 W-G：神职定位（God-Role Intel）

**问题**：在我目前掌握的信息下，场上哪些好人最像神职？优先刀谁 / 抗推谁？

**形状**：每个存活好人 j 一条记录（非 N×N）；**每位狼人独立维护一份**。

```text
W-G 神职定位（3号狼 · 仅本人可见）

目标 │ Seer  Witch Guard Hunter Villager │ threat │ priority
─────┼──────────────────────────────────┼────────┼────────────
 2号 │ 0.75   0.10  0.05   0.00   0.10  │  0.92  │ kill_tonight
 1号 │ 0.15   0.55  0.05   0.00   0.25  │  0.70  │ watch
 5号 │ 0.08   0.05  0.40   0.00   0.47  │  0.35  │ low
```

**威胁分（规则层可算）**：

```text
threat_score = w_seer·P(Seer) + w_witch·P(Witch) + w_guard·P(Guard) + w_lead·leadership
```

默认权重：预言家 > 女巫 > 守卫 > 猎人 > 村民。

**字段**：

```python
@dataclass
class GodRoleBelief:
    target_seat: int
    role_distribution: dict[str, float]
    threat_score: float
    priority: str          # kill_tonight | watch | low
    evidence: list[str]
    updated_round: int
    contributors: list[int]  # 哪些狼队友贡献过该条目
```

### 5.2 W-E：暴露雷达（Exposure Radar）

**问题**：哪些好人已在盯上**我**？被谁盯上？

**形状**：行 = **本人**（固定为 owner_seat），列 = 好人 j，格 = 「我认为 j 对我的怀疑度」。

```text
W-E 暴露雷达（3号狼 · 仅本人可见）

         │  1号    2号    5号    6号  │ overall │ stance
─────────┼───────────────────────────┼─────────┼──────────
（本人）  │  0.20   0.65   0.30   —    │  0.65   │ hide
```

> **隔离规则**：`exposure_radar` 增量中的 `wolf_seat` 必须等于提交者座位；忽略对队友座位的更新。运行时不再展示队友暴露行。

**行汇总 `WolfExposureProfile`**：

| 字段 | 说明 |
|------|------|
| `overall_exposure` | 行内 max 或加权聚合 |
| `top_suspectors` | 怀疑度最高的好人座位 |
| `suggested_stance` | `hide` / `push` / `bus` / `counter` / `sacrifice` |

**姿态建议（规则层）**：

| stance | 条件（示例） |
|--------|--------------|
| `hide` | overall < 0.35 |
| `push` | overall 低且 W-G 有可抗推目标 |
| `bus` | overall > 0.6 |
| `counter` | 被当前轮发言点名 |
| `sacrifice` | overall > 0.85，保队友 |

### 5.3 维护方式：私有增量 → 本人 merge

```text
白天 AFTER_SPEECH（observer 是狼）:
  LLM 输出 wolf_camp_delta
       → merge 进 GameState.wolf_camp_minds[observer_seat]

狼队夜间讨论 / mind state 采集:
  Prompt 注入 format_wolf_camp_board(本人面板)

merge 策略（保守）:
  W-G 同 target: 与本人上一帧加权平均
  W-E: 仅更新 wolf_seat == contributor_seat 的格子；同 observer 取 max
  公开跳身份: 硬约束对应列
  禁止: 将任一狼的 delta merge 进其他狼的面板
```

### 5.4 运行时存储

```python
@dataclass
class WolfCampMindModel:
    owner_seat: int
    wolf_seats: list[int]              # 恒为 [owner_seat]
    god_role_intel: dict[int, GodRoleBelief]       # target_seat -> ...
    exposure_radar: dict[int, WolfExposureProfile] # owner_seat -> ...
    revision: int
    history: list[dict]                            # 每轮 merge 快照
```

挂载位置：`GameState.wolf_camp_minds: dict[int, WolfCampMindModel]`（按狼座位索引）；`InformationHub` 持有同一引用，展示/merge 时经 `get_wolf_camp_mind(minds, player)` 解析。

## 6. 与投票意向的联合输出

### 6.1 设计原则

- **不新增**独立的「仅信念」LLM 调用轮次
- 在现有 `_collect_vote_intentions` 时机，扩展 Schema 为 **`MindStateDecision`**
- 发言仍用 **`SpeechDecision`**（含 `private_thought`）；信念在发言后听众更新

### 6.2 采集时机（与现网一致）

| anchor | 时机 | 采集对象 |
|--------|------|----------|
| `INITIAL` | 白天讨论开始前 | 全体存活听众 |
| `AFTER_SPEECH` | 每位发言者结束后 | 全体存活听众 |

参考：`InformationHub.run_roundtable()` → `_collect_vote_intentions()`。

### 6.3 统一 Schema（好人 / 通用）

```python
class MindStateDecision(BaseModel):
    """投票意向 + 一阶/二阶信念，一次 generate_response 提交。"""

    seat: int = Field(..., ge=0, description="投票意向座位，0=观望")
    reason: str | None = None

    first_order: list[BeliefEntry] = Field(default_factory=list)
    second_order: list[SecondOrderEntry] = Field(default_factory=list)
```

`BeliefEntry` / `BeliefMatrixDecision` 已在 `strategy/decisions.py` 预留；需新增 `SecondOrderEntry`。

### 6.4 狼人额外字段

```python
class WolfCampDelta(BaseModel):
    god_role_intel: list[GodRoleDelta] = Field(default_factory=list)
    exposure_radar: list[ExposureRadarDelta] = Field(default_factory=list)
```

狼听众的 `MindStateDecision` 可扩展为：

```python
class WolfMindStateDecision(MindStateDecision):
    wolf_camp_delta: WolfCampDelta | None = None
```

或单一模型内 `wolf_camp_delta: Optional[WolfCampDelta]`（狼以外恒为 null）。

### 6.5 Prompt 注入（上一帧信念）

每次采集意向前，在 `additional_context` 追加：

```text
【你上一帧信念摘要 · 仅自己可见】
一阶（狼概率）: 1→0.15, 2→0.20, 3→0.45, ...
二阶（他人疑我）: 2→0.65, 5→0.30, ...
请结合刚听完的发言更新信念，并给出最新投票意向。
```

狼听众额外追加 W 摘要（见 §5.2 Prompt 示例）。

### 6.6 完整 JSON 示例（5 号村民，AFTER_SPEECH）

```json
{
  "anchor": "after_speech",
  "round": 2,
  "observer_seat": 5,
  "speaker_seat": 2,
  "mind_state": {
    "seat": 3,
    "reason": "2号发言后我更怀疑3号",
    "first_order": [
      { "target_seat": 1, "wolf_probability": 0.15, "note": "用药节奏" },
      { "target_seat": 2, "wolf_probability": 0.20, "note": "带票像神职" },
      { "target_seat": 3, "wolf_probability": 0.45, "note": "前后矛盾" },
      { "target_seat": 4, "wolf_probability": 0.50, "note": "跟票异常" },
      { "target_seat": 5, "wolf_probability": 0.00, "note": "本人" },
      { "target_seat": 6, "wolf_probability": 0.00, "note": "已出局已知" }
    ],
    "second_order": [
      { "observer_seat": 2, "suspects_me_as_wolf": 0.25 },
      { "observer_seat": 3, "suspects_me_as_wolf": 0.40 }
    ]
  }
}
```

## 7. 规则层硬更新（BeliefUpdater）

LLM 输出后，由 **规则引擎** 校正，避免漂移与违反隔离。

### 7.1 硬约束事件

| 事件 | 影响范围 | 更新规则 |
|------|----------|----------|
| 玩家 j 死亡且身份公开 | 所有 Agent 的 B1[*][j] | one-hot 为真实 role |
| 预言家验 j（仅验人者） | 验人者 B1[me][j] | 狼/好 hard 0/1 |
| 狼人已知队友 | 狼的 B1[me][队友] | Werewolf = 1.0 |
| 女巫知刀口/用药 | 女巫私有 B1/W-G | 仅本人 |
| 跳预言家/女巫（公开） | 全场 W-G / 相关 B1 列 | Seer/Witch 概率上调 |

### 7.2 软约束（可选）

| 信号 | 规则 |
|------|------|
| j 投票给已知好人 | B1[*][j] 狼概率 +δ |
| j 投票给已知狼 | B1[*][j] 狼概率 −δ |
| 发言 swing 对齐 | 与 `VoteIntentionTracker` 联动 |

### 7.3 归一化

- 每列 `role_distribution` 求和归一化为 1
- `wolf_probability` clip 到 [0, 1]
- 对角线 self 固定为真实身份（若已知）

建议模块：`strategy/belief_updater.py`（纯函数，便于单测）。

## 8. 信息隔离与可见性

### 8.1 谁能看到什么

| 数据 | 可见范围 | 持久化 visibility |
|------|----------|-------------------|
| B1 / B2 行 | 仅该 Agent | `private` / 仅写入该 player 的 run 视图 |
| W-G / W-E | 仅该狼人本人（按 `owner_seat`） | `private` / 复盘 `god` 视图可按座位过滤 |
| belief_log 快照 | 引擎 / 复盘 / Coach | `god` / 赛后公开 |

### 8.2 与现有模块对齐

| 模块 | 关系 |
|------|------|
| `PlayerObservation` | Prompt 上下文仍基于可见事件；信念不泄露私密事件 |
| `EventVisibility` | 翻牌 / 死亡公布触发 B1 列塌缩 |
| `InformationHub` | 意向采集 + 信念更新钩子 |
| `SpeechDecision.private_thought` | 与 B2 互补；私有思维不广播，信念结构化存储 |

### 8.3 禁止事项

- 将其他玩家的 B1 行注入某 Agent Prompt
- 将 W 面板注入好人 Prompt
- 在公开发言 channel 广播信念矩阵

### 9.3 调用链（白天）

```text
run_roundtable
  ├─ INITIAL → _collect_mind_states(anchor=INITIAL)
  │              → invoke MindStateDecision
  │              → BeliefUpdater.apply
  │              → agent.belief_state = ...
  │              → [狼] merge wolf_camp_delta
  │
  for speaker in speakers:
      ├─ request_speech → SpeechDecision
      └─ AFTER_SPEECH → _collect_mind_states(anchor=AFTER_SPEECH)
                        → record SpeechMindRecord (before/after beliefs + swing)
```

### 9.4 与 `BeliefMatrixDecision` 迁移

现有 stub：

```python
class BeliefEntry(BaseModel):
    target_seat: int
    wolf_probability: float
    note: str | None

class BeliefMatrixDecision(BaseModel):
    beliefs: list[BeliefEntry]
```

**迁移策略**：保留 `BeliefEntry`；`BeliefMatrixDecision` 标记 deprecated，由 `MindStateDecision.first_order` 替代；避免两套并行 Schema。

## 14. 风险与对策

| 风险 | 对策 |
|------|------|
| LLM 信念不一致 / 幻觉 | 规则硬更新 + 上一帧注入 + clip 归一化 |
| Token 变长 | 首期仅 wolf_prob + top-3 可疑；完整分布二期 |
| O(N²) 调用已很多 | **禁止**独立信念轮次；必须与意向合并 |
| 狼人 W 被误泄露给其他狼/好人 | 按 `owner_seat` 隔离 + `get_wolf_camp_mind` + 集成测 |
| 与 Skill / 策略 Prompt 冲突 | 信念仅作「局势摘要」，不覆盖角色卡；Skill 经 `belief_signals` 子集匹配后作「对局经验」补充（见 agent_team/skills/README.md） |

## 15. Skill 信念匹配注入（2026-05-23）

信念矩阵除 Prompt 摘要外，驱动 **Skill 动态注入**（唯一注入方式）：

1. PostGame 从 `beliefs.jsonl` 快照自动生成 `belief_signals`，写入 Skill MD frontmatter。
2. 每次私密 LLM 决策前（发言、选座、女巫、投票意向等）由 `InformationHub._refresh_actor_belief_skills` → `refresh_player_belief_skills()` 检测当前 signal，匹配 active Skill 子集。
3. 匹配结果注入 decision context（`【信念匹配的对局经验 · 仅供参考】`），不替代 B1/B2 摘要。

测试：`tests/agent_team/test_skill_markdown.py`（12p 对局 4/4 回匹配）；`tests/agent_team/test_information_hub_belief_skills_refresh.py`（私密决策 refresh 顺序）。
