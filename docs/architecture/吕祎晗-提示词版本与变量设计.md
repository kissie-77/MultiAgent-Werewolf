# 提示词与 Skill 版本控制设计

> **读者**：Prompt / Agent / 评测队友
> **关联**：[strategy/DESIGN.md](../strategy/DESIGN.md)、[prompt_tuning.md](./prompt_tuning.md)、[evaluation/DESIGN.md §12](../evaluation/DESIGN.md)
> **原则**：运行时按**身份**分包；**每次更新 bump 新版本目录**；默认加载**最新**；评测合并走 evolution，不在 JSON 里直接覆盖运行时

---

## 1. 要解决的问题

| 旧现状 | 问题 |
|--------|------|
| 全量 Prompt 打在 `prompts/v2/` 一个目录 | 改一个身份也要动整包；无法按角色独立进化 |
| 全局 `prompt_version: v2` | 无法做到「狼人 v3、预言家仍 v1」 |
| Skill 平铺在 `skills/<role>/` | 写回覆盖旧卡；无法回溯对比 |
| 固定默认 v1 | 进化 bump 后运行时仍读旧版 |

**目标**：

1. **Prompt**：每身份独立目录 `prompts/roles/<role>/<version>/role.yaml` + 共享 `agent_base.md`
2. **Skill**：`agent_team/skills/<role>/<skill_version>/*.md`
3. **默认**：未 pin 的身份自动用磁盘**最新**版本
4. **进化 / 回放**：`RoleVersionManifest` 记录每身份 pin；`version_manifest.json` 可恢复

## 2. 目录结构

### 2.1 Prompt（内置 + 进化产物）

```text
src/llm_werewolf/strategy/prompts/
  shared/
    agent_base.md
  roles/
    wolf/
      v1/role.yaml
      v1/manifest.yaml
      v2/...                    # 进化 bump
    prophet/v1/...
    ...                         # 共 22 个 prompt_role_key

artifacts/prompt_roles/         # prompt_evolver 写入的生成包（与内置合并检索）
  wolf/v2/
```

### 2.2 Skill

```text
src/llm_werewolf/agent_team/skills/
  wolf/
    v1/*.md
    v2/*.md                     # PostGame 库写入前 bump
  prophet/v1/...
```

PostGame 写库流程：先 `next_skill_version()` → 复制旧版 MD 到新目录 → 追加新卡 → 更新 manifest 中该身份的 `skill_versions`。

## 3. RoleVersionManifest

```python
@dataclass
class RoleVersionManifest:
    default_prompt_version: str = "latest"
    default_skill_version: str = "latest"
    prompt_versions: dict[str, str] = field(default_factory=dict)
    skill_versions: dict[str, str] = field(default_factory=dict)
```

| 场景 | 解析规则 |
|------|----------|
| 普通开局 | 各身份 `prompt_version_for` / `skill_version_for` → 扫描目录取 **max(vN)** |
| 进化 round N+1 | 读上一轮 `version_manifest.json` → `restore_runtime_from_manifest()` |
| 显式 pin 某身份 | `prompt_versions["wolf"] = "v2"` 强制该身份用 v2 |
| 目录为空 | fallback `v1`（或 manifest 的 default，若为具体 vN） |

全局 active manifest：`set_active_manifest()` / `get_active_manifest()`，bootstrap 与 `PlayersConfig.role_version_manifest()` 注入。

### 4.3 进化

from llm_werewolf.evaluation.evolution.prompt_evolver import evolve_prompt_from_run

## 5. 配置

| 层级 | 字段 | 默认 | 说明 |
|------|------|------|------|
| YAML | `prompt_version` | `latest` | 遗留字段；同步到 `role_versions.default_prompt_version` |
| YAML | `role_versions.prompt_versions` | `{}` | 可选：按身份 pin |
| YAML | `role_versions.skill_versions` | `{}` | 可选：按身份 pin |
| PostGame / 进化 | `version_manifest.json` | — | schema `agent_version_manifest_v2`，含 `role_version_manifest` |

示例（pin 狼人到 v2，其余跟 latest）：

```yaml
role_versions:
  prompt_versions:
    wolf: v2
  skill_versions:
    wolf: v2
```

## 6. 与评测闭环的衔接

| 产物 | 版本语义 |
|------|----------|
| `prompt_proposals.json` | `prompt_role_key` + patch；evolver 按身份 bump |
| `applied_prompt_proposals.json` | 含 `role_version_manifest`、`changed_prompt_roles` |

## 8. 队友 FAQ

**Q：修狼人 Prompt 改哪个文件？**  
A：`strategy/prompts/roles/wolf/<最新或目标版本>/role.yaml`；共用行为改 `prompts/shared/agent_base.md`。

**Q：新版本怎么开？**  
A：复制 `wolf/v1/` → `wolf/v2/` 再改内容；或跑 PostGame/evolution 自动 bump。不要覆盖旧目录。

**Q：运行时怎么知道用哪版？**  
A：默认各身份自动用目录里**最新** `vN`；进化 manifest 或 YAML `role_versions` 可 pin。

**Q：Skill 写回会覆盖旧卡吗？**  
A：不会。写回前先 `next_skill_version()`，新 MD 进新文件夹；旧版保留供 A/B 与回放。
