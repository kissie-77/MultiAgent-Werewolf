# 提示词版本与变量化设计

> **读者**：Prompt / Agent / 评测队友
> **关联**：[工程架构重构计划.md](./%E5%B7%A5%E7%A8%8B%E6%9E%B6%E6%9E%84%E9%87%8D%E6%9E%84%E8%AE%A1%E5%88%92.md)、[prompt_tuning.md](./prompt_tuning.md)、[吕祎晗-评测模块优化设计.md](./%E5%90%95%E7%A5%8E%E6%99%97-%E8%AF%84%E6%B5%8B%E6%A8%A1%E5%9D%97%E4%BC%98%E5%8C%96%E8%AE%BE%E8%AE%A1.md)
> **原则**：运行时 **只保留 v2+**；代码里只引用 **版本化变量名**；正文在 **独立文件**，便于替换与 A/B；**不**在评测 JSON 里直接覆盖运行时（仍走 draft 流程）

---

## 1. 要解决的问题

| 现状                                                    | 问题                                                              |
| ------------------------------------------------------- | ----------------------------------------------------------------- |
| 正文集中在 `strategy/role_prompts.py`                   | 改 Prompt = 改 Python，难 diff、难给非开发同事替换                |
| `PromptManager` 直接读类常量                            | 无版本切换；评测产出的 `prompt_proposals.json` 无法对应到稳定槽位 |
| Demo 轨 `system.py` 与 AgentScope 轨 `RolePrompts` 双轨 | 变量命名不统一                                                    |

**目标**：

1. 系统/角色/阶段 Prompt 通过 **变量名** 引用，例如 `v2.agent.base`、`v2.role.wolf.instruction`。
2. **版本号** 嵌入变量名前缀（`v2.`、`v3.`），切换版本 = 切换 registry，而非改代码。
3. **真实文案** 放在 `strategy/prompts/<version>/` 下的 `.md` / `.yaml`，与代码分离。
4. 评测模块产出的 Prompt 补丁 / Skill，引用 **变量 id**，便于合并与对比文档。

## 2. 变量命名规范

### 2.1 格式

```text
{version}.{domain}.{name}[.{field}]
```

| 段        | 含义              | 示例                                         |
| --------- | ----------------- | -------------------------------------------- |
| `version` | 主版本，小写 `v2` | `v2`                                         |
| `domain`  | 域                | `agent` / `role` / `phase` / `plan` / `demo` |
| `name`    | 实体              | `base`、`wolf`、`vote_begin`                 |
| `field`   | 可选字段          | `instruction`、`suggestion`、`role_name`     |

### 2.2 域划分

| domain  | 用途                                | 消费方                                       |
| ------- | ----------------------------------- | -------------------------------------------- |
| `agent` | AgentScope 角色策略卡骨架           | `PromptManager.build_role_strategy_prompt`   |
| `role`  | 各身份卡片字段                      | 填入 `agent.base` 的 `{role_*}` 槽位         |
| `phase` | 阶段/流程文案（原 `GamePrompts`）   | `bridge`、`role_night_plans`、`voting_phase` |
| `plan`  | 策略计划文本（原 `PlanStrategies`） | `resolve_plan_text`                          |
| `demo`  | DemoAgent 轨 `SYSTEM_PROMPT`        | `prompts/system.py` 迁移目标                 |

### 2.3 运行时槽位（与模板 `{...}` 对应）

`v2.agent.base` 模板内保留 Python `str.format` 槽位：

| 槽位                 | 来源                             |
| -------------------- | -------------------------------- |
| `{number}`           | 座位号                           |
| `{role_name}`        | `v2.role.{key}.role_name`        |
| `{role_instruction}` | `v2.role.{key}.role_instruction` |
| `{suggestion}`       | `v2.role.{key}.suggestion`       |
| `{plan}`             | `PlanStrategies` / `v2.plan.*`   |

## 4. 代码 API（`prompt_registry.py`）

```python
registry = PromptRegistry.default()  # 默认加载 active 版本 v2

registry.get_text("v2.agent.base")  # 原始模板字符串
registry.get_role_card("v2.role.wolf")  # dict[str, str]
registry.resolve("v2.agent.base", **slots)  # format 后全文

registry.list_variables(version="v2")  # 评测 / 文档生成
registry.variable_path("v2.role.wolf")  # 磁盘路径，供 diff / 提案定位
```

**`PromptManager` 变更（最小）：**

- `build_role_strategy_prompt(..., prompt_version: str = "v2")`
- 内部：`registry.get_text(f"{prompt_version}.agent.base").format(...)`
- `get_role_strategy_config(key)` → `registry.get_role_card(f"{prompt_version}.role.{key}")`

**`role_prompts.py`：** 保留 `RolePrompts` / `GamePrompts` 类名，属性在 import 时从 registry 填充，避免全仓库改 import。

## 5. 版本切换与配置

| 层级     | 字段                                                     | 默认 |
| -------- | -------------------------------------------------------- | ---- |
| YAML     | `prompt_version: v2`（`PlayersConfig`）                  | `v2` |
| PostGame | `prompt_version`（已有）                                 | `v2` |
| 比赛     | 显式指定 `v3` 目录存在且 `manifest.status=active` 时启用 |      |

**规则：**

- 运行时 **不加载** `v1`；历史仅文档/archive。
- `v3` 等新版本 = **新目录** + 新变量前缀，旧版文件 **保留只读**。
- 评测 `prompt_proposals.json` 中 `target_variable: "v2.role.prophet.suggestion"` 指向合并目标。

## 6. 与评测闭环的衔接

| 评测产物                       | 引用方式                                                  |
| ------------------------------ | --------------------------------------------------------- |
| `prompt_proposals.json`        | `target_variable` + `action: append\|replace` + `text_zh` |
| `role_skills.json`             | 可选 `related_variables: ["v2.role.wolf.suggestion"]`     |
| `prompt_comparison.md`（后续） | 按 variable id diff 两版本文件                            |

合并 Prompt（人工或脚本）**只改** `strategy/prompts/v3/...` 文件，不改 Python。

**PostGame roster**：Skill 的 `prompt_role_key` 依赖 `RunContext.roster`。离线复盘须从 `events.jsonl` 全量事件（`role_acting` / `player_discussion` / `role_revealed` / `player_eliminated`）补全身份，不能只靠出局事件；详见 [评测模块优化设计 §6.3](./%E5%90%95%E7%A5%8E%E6%99%97-%E8%AF%84%E6%B5%8B%E6%A8%A1%E5%9D%97%E4%BC%98%E5%8C%96%E8%AE%BE%E8%AE%A1.md) 与 `evaluation/post_game/run_context.py`。

## 9. 队友 FAQ

**Q：修 Prompt 改哪个文件？**
A：`strategy/prompts/v2/roles/wolf.yaml` 或 `text/agent_base.md`，不要改 `manager.py`。

**Q：新版本怎么开？**
A：复制 `v2/` → `v3/`，改 `manifest` 与变量前缀为 `v3.`，YAML 指定 `prompt_version: v3`。

**Q：变量名会出现在 Agent 消息里吗？**
A：不会；变量名仅代码/registry 内部使用，Agent 只见 format 后的中文。
