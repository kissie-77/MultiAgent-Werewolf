# 工程架构重构计划

## 目标

本计划用于把项目整理为清晰的工程职责模块。队友已确认旧路径不再需要，因此本次重构不再保留过渡入口，后续开发统一走新的模块路径。

| 目标模块       | 职责                                                        | 当前主路径                                                           |
| -------------- | ----------------------------------------------------------- | -------------------------------------------------------------------- |
| `game_runtime` | 狼人杀规则、角色、动作、状态、流程调度、配置、胜负判定      | `src/llm_werewolf/game_runtime/`                                     |
| `agent_team`   | AgentScope 多智能体运行、Agent 创建、消息隔离、LLM 调用封装 | `src/llm_werewolf/agent_team/`                                       |
| `strategy`     | Prompt、角色策略、结构化决策、阶段输出契约、投票意向        | `src/llm_werewolf/strategy/`                                         |
| `interface`    | 启动入口、模式选择、CLI/TUI/评估命令                        | `src/llm_werewolf/interface/`                                        |
| `ui`           | 对局界面、玩家面板、聊天面板、人机交互组件                  | `src/llm_werewolf/ui/`                                               |
| `evaluation`   | 评估、复盘、bad case、调优闭环                              | `src/llm_werewolf/evaluation/`、`docs/architecture/prompt_tuning.md` |

## 当前关键决策

### 1. AgentScope 主线

- 唯一 AgentScope 玩家主实现为 `src/llm_werewolf/agent_team/agentscope_agent.py`。
- Agent 创建统一走 `src/llm_werewolf/agent_team/factory.py`。
- 消息隔离、消息路由、可见性通道和串行调用保护统一放在 `agent_team`。
- 旧 AgentScope 入口已删除，后续不再维护两套 AgentScope 实现。

### 2. Prompt 主线

- 角色策略 Prompt 以 `src/llm_werewolf/strategy/role_prompts.py` 为准。
- AgentScope 角色绑定统一通过 `src/llm_werewolf/game_runtime/prompts/manager.py` 中的 `PromptManager` 构建最终系统 prompt。
- `agent_team/factory.py` 与 `agent_team/agentscope_agent.py` 不再直接拼接角色 Prompt，只调用统一 PromptManager。
- 后续 Prompt 调优记录写入 `docs/architecture/prompt_tuning.md`，不把调优依据散落在代码注释里。

### 3. Strategy 主线

- `src/llm_werewolf/strategy/decisions.py` 是结构化决策模型主实现。
- `src/llm_werewolf/strategy/phase_outputs.py` 是阶段输出契约主实现。
- `src/llm_werewolf/strategy/vote_intention.py` 是投票意向追踪与说服效果记录主实现。
- `strategy` 只负责“Agent 怎么想、怎么表达、怎么形成决策”，不承接游戏规则状态。

### 4. Game Runtime 主线

- `src/llm_werewolf/game_runtime/` 是狼人杀规则与运行时主实现包。
- 运行时内部负责角色、动作、状态、事件、引擎、配置、胜负判定和序列化。
- 源码内部已统一使用 `llm_werewolf.game_runtime.*`。
- 运行时通过 `PhaseInteraction` 表达“需要一次玩家决策”，由 `agent_team` 负责具体 Agent 调用。

### 5. Interface 与 UI

- `src/llm_werewolf/interface/` 负责命令入口和模式选择。
- `src/llm_werewolf/ui/` 保持独立，专注展示与交互组件。
- 模式选择放在入口层，后续可承接全 Agent、人机交互、基础局、警徽流和扩展角色等模式。

## 后续注意事项

- 新代码只引用新的六个主模块路径。
- 不再新增旧路径兼容文件。
