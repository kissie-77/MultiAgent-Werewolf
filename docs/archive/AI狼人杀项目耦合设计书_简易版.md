# AI狼人杀项目耦合设计书（简易版）

## 一、项目定位

- **目标**：AI比赛项目
- **核心**：快速开发、Agent智能表现、功能完整
- **不追求**：生产级基础设施

---

## 二、技术选型

| 组件           | 技术            | 职责                         |
| -------------- | --------------- | ---------------------------- |
| **游戏引擎**   | LLMWereWolf     | 阶段流转、胜负判定、事件日志 |
| **智能体框架** | AgentScope      | AI推理、信息隔离、记忆管理   |
| **大语言模型** | OpenAI/通义千问 | 智能决策                     |

---

## 三、核心耦合点

### 1. 接口统一：字符串 → Msg对象

| 项目         | 说明                                                 |
| ------------ | ---------------------------------------------------- |
| **当前问题** | LLMWereWolf期望字符串输入输出，AgentScope使用Msg对象 |
| **修改方案** | 修改LLMWereWolf的AgentProtocol接口，直接使用Msg      |
| **修改理由** | 减少转换开销，保留完整信息，简化代码                 |
| **工程量**   | 2.5天                                                |
| **修改文件** | `protocols.py`、各Phase文件                          |

### 2. 同步 → 异步

| 项目         | 说明                                    |
| ------------ | --------------------------------------- |
| **当前问题** | LLMWereWolf是同步调用，AgentScope是异步 |
| **修改方案** | LLMWereWolf所有Phase方法改为async/await |
| **修改理由** | MsgHub必须异步使用，信息隔离是核心需求  |
| **工程量**   | 2-3天                                   |
| **修改文件** | 所有engine/\*.py文件                    |

### 3. 信息隔离：prompt控制 → MsgHub

| 项目         | 说明                                                |
| ------------ | --------------------------------------------------- |
| **当前问题** | LLMWereWolf通过prompt手动控制信息可见性             |
| **修改方案** | 使用AgentScope的MsgHub自动广播                      |
| **修改理由** | 原生支持信息隔离，代码更简洁                        |
| **工程量**   | 2天                                                 |
| **修改文件** | `night_phase.py`、`day_phase.py`、`voting_phase.py` |

### 4. 决策解析：文本 → 结构化输出

| 项目         | 说明                                   |
| ------------ | -------------------------------------- |
| **当前问题** | LLMWereWolf用正则解析文本决策          |
| **修改方案** | 使用AgentScope的结构化输出（Pydantic） |
| **修改理由** | 稳定可靠，便于引擎处理                 |
| **工程量**   | 1-2天                                  |
| **修改文件** | 新增models.py，修改各Phase调用         |

### 5. 记忆机制：字符串列表 → Memory模块

| 项目         | 说明                                      |
| ------------ | ----------------------------------------- |
| **当前问题** | LLMWereWolf用简单字符串列表管理记忆       |
| **修改方案** | 由AgentScope的Memory模块自动管理          |
| **修改理由** | 功能完整，与Agent集成紧密                 |
| **工程量**   | 0.5天（基本不需要改，AgentScope自动处理） |
| **修改文件** | 无                                        |
