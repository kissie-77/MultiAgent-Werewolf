# 前端规划

> **用途**：供 UI/UX 与前端对接后端，明确每页数据、交互与接口。
> **后端**：FastAPI 页面 API + 写操作 API 已实现（`werewolf-api`）。
> **前端**：`frontend/` 已有 React 19 + Three.js 原型（Express + Gemini 临时后端，见 `frontend/README.md`）；**待迁移**为对接本文档中的 `/api/v1/pages/*` 与 `/api/v1/games/*`。
> **更新**：2026-05-23

## 1. 技术约定

### 1.0 现有前端与目标架构

| 层级 | 现状 | 目标 |
|------|------|------|
| 目录 | `frontend/`（Vite + React + Three.js） | 保留并扩展为全站 14 页 |
| 开发启动 | `cd frontend && npm run dev` → 端口 3000 | 开发期可 proxy 到 `werewolf-api :8000` |
| 对局 API | Express `/api/game/*`（`store.ts`） | 改为 `POST /games/start` + `GET /games/{id}/status` 轮询 |
| 静态页数据 | 前端硬编码 / Gemini | `GET /api/v1/pages/*` 整页拉取 |

迁移时优先改 `frontend/src/store.ts`：把 `/api/game/reset` 等映射到本文档 §1.4、§1.6 的 FastAPI 契约；3D 场景组件可复用。

### 1.1 启动与基础地址

```bash
uv run werewolf-api --host 127.0.0.1 --port 8000
```

| 接口 | 作用 |
|------|------|
| GET /health | 健康检查 |
| GET /api/v1/pages | 页面 API 路径索引 |
| GET /api/v1/pages/spec | 全部页面字段说明（机器可读） |
| GET /api/v1/pages/spec/{page_key} | 单页字段说明 |
| GET /api/v1/actions/spec | 全部写操作说明（机器可读） |
| GET /api/v1/games/modes | 可选开局模式与 config 预设列表 |

### 1.2 响应格式

统一包裹为 `ApiResponse`：

```json
{ "success": true, "data": {}, "message": null }
```

### 1.3 数据获取策略

- **整页渲染**：优先 `GET /api/v1/pages/*`，一次拿齐当页所需数据。
- **Legacy 只读**：`/api/v1/home`、`/game`、`/content/*`、`/roles/*`、`/models/*`、`/runs/*`、`/replay/*` 仍可用，新前端建议只用 pages + actions。
- **列表分页**：`GET /api/v1/runs?page=&page_size=&source=runs|eval`。

### 1.4 写操作与观战轮询

前端**不需要**自己拼开局逻辑，调用 POST 即可：

| 场景 | 方法 | 接口 |
|------|------|------|
| 开局模式列表 | GET | /api/v1/games/modes |
| 开局 | POST | /api/v1/games/start |
| 观战轮询 | GET | /api/v1/games/{run_id}/status |
| 取消后台对局 | POST | /api/v1/games/{run_id}/cancel |
| 生成 PostGame | POST | /api/v1/runs/{run_id}/post-game |
| 模型对比提交 | POST | /api/v1/models/compare |

- **无 WebSocket**：对局进行中，游戏页定时轮询 `status` + 可选刷新 `pages/game`。
- **source 参数**：`runs`（默认，artifacts/runs）或 `eval`（评测批次）。

### 1.6 开局 API（POST /api/v1/games/start）

所有 body 字段均为**可选**；不传任何字段时默认 `all_agent` + `basic` → `demo-6`（无需 API Key）。

#### 选板子 / 模式

| 方式 | 示例 | 说明 |
|------|------|------|
| 空 body | `{}` | 默认 demo-6 |
| `config_id` | `llm-12p-doubao` | 选 configs YAML |
| `participation` + `rules` | `all_agent` + `badge_flow` | 内置模式，见 GET `/games/modes` |

内置 rules：`basic` / `badge_flow` / `extended_roles`。常用 config_id：`demo-6`、`human-6p-demo`、`llm-6p-doubao`、`llm-12p-doubao`、`agentscope`。

#### 自定义人数 / 姓名 / 模型

| 字段 | 说明 |
|------|------|
| `player_count` | 6-20 人 |
| `defaults` | 全局 model / base_url / api_key_env |
| `players[]` | 按座位索引覆盖 |
| `human_seats` | 1-based 人类座位，如 [1] |
| `badge_flow` | true 开启警长/警徽流 |

API Key 不写 YAML：用 `api_key_env` 指向服务端 `.env`。有 roster 定制时写入 `launch_roster.json`。

响应额外返回：participation / rules / player_count / human_seats / badge_flow / custom_roster。

## 2. 逐页功能说明

以下每节包含：**当页可获取数据**、**当页可实现功能**（通俗）、**接口简要**。

### 2.1 首页 `/`

**当页可获取数据**
- hero：主标题、副标题、CTA 文案与跳转
- stats_cards：对局数、角色数、配置数、含复盘局数
- recent_runs：最近 8 局摘要（run_id、人数、胜负、是否有 PostGame/复盘）
- quick_actions：快捷入口按钮
- game_modes：可选开局模式（含 config_id）
- nav_links：全站导航

**当页可实现功能**
- 展示平台概览与数据卡片
- 点击最近对局 → 复盘或观战
- 点击快捷入口 / 模式卡 → 跳转游戏页并预选 config_id
- **开始新局**：选模式后 POST 开局，跳转 `/game?run_id=...`

**接口简要**
- GET `/api/v1/pages/home` — 整页数据
- GET `/api/v1/games/modes` — 开局模式列表
- POST `/api/v1/games/start` — 开局（模式 + roster 定制）
- GET `/api/v1/actions/spec` — 查 action 契约

---

### 2.2 主游戏页 `/game`

**当页可获取数据**
- modes[]：6p/12p/demo 等配置卡（含 config_id）
- active_run：当前 run 元数据、产物标记
- snapshot：阶段、回合、存活/死亡、是否结束
- board_preset：板子人数、角色、timeout
- players[]、camp_counts：座位与阵营统计
- recent_events[]：最近约 20 条事件

Query：`run_id`+`source` 加载指定局；`config_id` 预选板子；无参仅返回 modes。

**当页可实现功能**
- 选择模式并开始新局（POST）
- 观战进行中对局：轮询 status，刷新快照与事件流
- 查看玩家座位、当前阶段、阵营存活
- 跳转复盘页、取消进行中对局
- 对局结束后提示「去看复盘」

**接口简要**
- GET `/api/v1/pages/game?run_id=&source=&config_id=` — 整页快照
- GET `/api/v1/games/modes` — 模式列表
- POST `/api/v1/games/start` — 开局（可带 roster 定制）
- GET `/api/v1/games/{run_id}/status?source=` — 轻量轮询（status + snapshot）
- POST `/api/v1/games/{run_id}/cancel` — 取消后台任务

---

### 2.3 复盘 `/replay/{run_id}`

**当页可获取数据**
- run：元数据、roster、分享信息
- timeline[]：全事件时间线
- phase_summary[]：按回合/阶段汇总
- turning_points[]：关键转折
- mvp_ranking[]、scores[]：MVP 与多维评分
- report_markdown、coach_excerpt：赛后报告
- belief_snapshots[]、belief_heatmap：信念矩阵 B1/B2
- wolf_camp_snapshots[]：狼队共享信念 W
- views_available[]：PostGame 多视角 Tab

**当页可实现功能**
- 按 Tab 浏览：时间线 / 评分 / 信念 / 报告 / 多视角
- 按回合、阶段筛选事件
- 分享 → 跳转 share 页
- 若 PostGame 缺失，点按钮触发生成

**接口简要**
- GET `/api/v1/pages/replay?run_id=&source=` — 整页（含信念字段）
- GET `/api/v1/replay/{run_id}` — Legacy，字段等价
- POST `/api/v1/runs/{run_id}/post-game` — 生成/重跑 PostGame

### 写操作请求体摘要

**GET /api/v1/games/modes** — 无 body，返回 modes / default_participation / default_rules。

**POST /api/v1/games/start** — 全部字段可选：

```json
{
  "config_id": "demo-6",
  "participation": "all_agent",
  "rules": "basic",
  "player_count": 8,
  "human_seats": [1],
  "badge_flow": true,
  "defaults": {
    "model": "gemini-3.1-flash",
    "base_url": "https://api.example.com/v1",
    "api_key_env": "WEREWOLF_API_KEY"
  },
  "players": [{ "name": "P1" }, { "name": "P2" }]
}
```

示例：`{}` · `{ "rules": "badge_flow" }` · `{ "config_id": "human-6p-demo" }`

**POST /api/v1/runs/{run_id}/post-game**
```json
{ "source": "runs", "force": false }
```

**POST /api/v1/models/compare**
```json
{ "ids": ["demo", "other-model"] }
```
