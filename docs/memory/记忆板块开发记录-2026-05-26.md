# 记忆板块开发记录

日期：2026-05-26

## 核心决策：ReMe 下线，Skill 接管语义记忆

**原因**：ReMe 依赖向量嵌入（embedding）+ 外部 API，增加部署复杂度和 API 成本。
队友已实现完整的 Skill 提取-存储-加载闭环（`evaluation/post_game/skill_extractor.py` + `agent_team/skill_loader.py`），
语义记忆直接复用 Skill `.md` 文件即可，无需额外向量层。

**变更**：
- `MemoryConfig` 删除所有 `reme_*` 字段
- `MemoryManager` 去掉 `ReMeSemanticBackend` 初始化路径
- `factory.py` 删除 `REME_*` 环境变量注入
- `.env.example` 清理 ReMe 配置段
- `reme_backend.py` 中 ReMe 向量存储代码标记废弃，保留 `LLMCompressor`（纯 LLM 调用，不依赖向量）

**语义记忆新方案**：`SemanticMemory` 调用 `skill_loader.load_role_skills()` 读取 `skills/<role>/*.md`，
按 frontmatter 的 `weight` 字段降序排序，取 top_k 注入 Agent prompt。

## 架构状态

```text
src/llm_werewolf/
├── evaluation/post_game/
│   ├── coach/
│   │   ├── __init__.py
│   │   └── coach.py                   # 占位，暂不实现
│   └── skill_generation/              # 局末 Skill 生成
├── agent_team/
├── memory/
│   ├── working_memory.py          # 工作记忆（规则式压缩，可插拔 compressor）
│   ├── episodic_memory.py         # 情景记忆（全局事件 + 玩家视角过滤）
│   ├── semantic_memory.py         # 语义记忆 → 调用 skill_loader 读 Skill .md
│   ├── procedural_memory.py       # 程序记忆（PromptManager 封装）
│   └── memory_manager.py          # 统一调度
├── skills/
│   └── <role>/*.md                # 局末由 PostGame 自动生成
└── skill_support/
    └── skill_loader.py            # 运行时加载 Skill MD（按 weight 降序）
```
