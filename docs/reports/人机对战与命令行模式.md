# 人机对战 / 可配置人数 / 警徽流 —— 命令行模式

> 本文档记录一次功能增量：在**尽量小、且追加式（不破坏现有逻辑、不改对局日志格式）**的前提下，
> 通过命令行参数支持三种新玩法。默认不带任何新参数时，行为与改动前**逐字节一致**。

## 一、新增能力一览

| 命令行参数 | 作用 | 默认 |
|---|---|---|
| `--human_seat 1` 或 `--human_seat 1,3` | 指定 1-based 座位为**人类玩家**（可多座位，逗号分隔） | 无（纯 Agent 局） |
| `--players N` | 覆盖**总座位数（含人类）**，范围 6–20 | 沿用 YAML 名单 |
| `--badge_flow` / `--nobadge_flow` | 开 / 关**警长·警徽流**（首夜后警长选举） | 关（与现状一致） |

这些参数对 `llm-werewolf`（控制台）与 `werewolf-tui`（TUI）两个入口都生效（python-fire 自动暴露）。

人类玩家**只需输入数字和中文文本**：

| 场景 | 输入 |
|---|---|
| 选目标 / 投票 / 用技能选人 | 座位号，如 `3`；可跳过时填 `0` |
| 是 / 否（如是否竞选警长） | `1` = 是，`0` = 否 |
| 白天 / 竞选发言 | 直接打中文（完整句子，≥15 字） |
| 女巫 | 救人输入 `救`；毒人输入 `毒 3`；不行动输入 `0` 或回车 |
| 丘比特等多选 | 两个不同座位号，空格分隔，如 `3 5` |

非法或不完整的输入会**就地提示并重试**，不会静默落到随机兜底。

## 二、如何启动

> **Windows 本机注意**：每次运行请前置 `PYTHONUTF8=1 PYTHONIOENCODING=utf-8`，
> 否则 GBK 控制台渲染 emoji 会报 `UnicodeEncodeError`。Linux/macOS 通常无需。

### 1. 纯 LLM 对战（所有玩家都是 AI）

```bash
# 6 人 DeepSeek
uv run llm-werewolf --config configs/llm-6p-deepseek.yaml

# 9 人 Doubao（豆包）
uv run llm-werewolf --config configs/llm-9p-doubao.yaml

# 加警徽流；并临时改成 8 人
uv run llm-werewolf --config configs/llm-6p-deepseek.yaml --players 8 --badge_flow
```

### 2. 人机对战（人类 + AI）

```bash
# 你坐 1 号，其余 5 个 DeepSeek 当对手（最完整的体验）
uv run llm-werewolf --config configs/llm-6p-deepseek.yaml --human_seat 1

# 你 + 真 AI，扩到 9 人，并开警徽流
uv run llm-werewolf --config configs/llm-9p-doubao.yaml --players 9 --human_seat 1 --badge_flow

# 多个真人（1 号和 3 号都是人）
uv run llm-werewolf --config configs/llm-6p-deepseek.yaml --human_seat 1,3
```

### 3. 离线人机对战（无需任何 API Key，机器人为占位 demo）

```bash
uv run llm-werewolf --config configs/human-6p-demo.yaml            # 你 + 5 个 demo 机器人
uv run llm-werewolf --config configs/human-6p-demo.yaml --players 9 --badge_flow
```

> `demo` 机器人的发言是占位文本，仅用于无 API 时验证流程（夜间技能、发言、投票、警长选举均可走通）；
> 真正“有意义的对话”需要用真实 LLM 对手（见上文 2）。

### Windows 完整示例

```bash
PYTHONUTF8=1 PYTHONIOENCODING=utf-8 uv run llm-werewolf \
  --config configs/llm-6p-deepseek.yaml --human_seat 1 --badge_flow
```

## 三、是否需要配置 API Key？

**取决于对手是不是真 LLM：**

| 玩法 | 是否需要 API Key |
|---|---|
| 纯 LLM 对战（deepseek / doubao / openai…） | **需要** |
| 人类 + 真 LLM（`--human_seat` 配 LLM 配置） | **需要**（给 AI 用） |
| 离线人机（`human-6p-demo.yaml`，对手是 `demo`） | **不需要** |
| 纯 demo（`demo-6.yaml`） | **不需要** |

配置方式（YAML 用 `api_key_env` 引用环境变量名，**密钥只写进 `.env`，不要写进 YAML**）：

1. 复制模板：`cp .env.example .env`
2. 在 `.env` 填入对应变量，例如：
   ```bash
   DEEPSEEK_API_KEY=sk-xxxxxxxx
   ARK_API_KEY=ark-xxxx-...-xxxx      # 火山方舟/豆包；注意带前后缀的完整串
   AGENT_SERIAL_DELAY_SECONDS=1.0     # 串行调用间隔，缓解 429 限流
   ```
3. YAML 里每个玩家用 `api_key_env: DEEPSEEK_API_KEY` 指向变量名即可。

> 本仓库的 `.env` 当前已配置 `DEEPSEEK_API_KEY` 与 `ARK_API_KEY`，纯 LLM / 人机（真 AI）可直接运行。
> 人类座位（`model: human`）和 demo 座位（`model: demo`）**不需要** `base_url` / `api_key`。

## 四、实现说明（供后续维护 / 合并参考）

### 设计原则
追加式、低合并冲突：**新增文件为主**，共享文件只加可选参数与少量调用行；新参数全部缺省，
默认路径与改动前一致；人类输入解析复用引擎已有的**非结构化文本兜底链路**，不改 `bridge.py` / 不改决策协议。

### 新增文件
| 文件 | 说明 |
|---|---|
| `src/llm_werewolf/agent_team/human_interactive_agent.py` | `HumanInteractiveAgent`：识别当前决策类型→给针对性提示→校验+归一化人类输入→交引擎解析；`input()` 用 `asyncio.to_thread` 包裹避免阻塞事件循环；EOF/中断安全降级 |
| `src/llm_werewolf/interface/player_count.py` | `resize_players_config`：按 `--players N` 扩/缩在内存中的名单（克隆非 human 座位 / 拒绝丢弃 human 座位），经 `model_validate` 重建以触发校验 |
| `src/llm_werewolf/interface/cli_overrides.py` | `parse_seat_list` + `apply_human_seats`：把 `--human_seat` 指定座位改成 `model="human"`；兼容 fire 把 `1,3` 解析成 tuple 的行为 |
| `configs/human-6p-demo.yaml` | 离线人机对战示例配置 |

### 追加式改动（共享文件）
| 文件 | 改动 |
|---|---|
| `agent_team/base.py` | `create_agent` 的 `model=="human"` 分支改为返回 `HumanInteractiveAgent`（保留旧 `HumanAgent` 兼容） |
| `interface/cli.py` / `interface/tui.py` | `main`/`_run_main` 增加 `players` / `human_seat` / `badge_flow` 三个可选参数；加载配置后应用覆盖；`prepare_game_roster` 后用 `game_config.model_copy(update={"enable_sheriff": True})` 切换警徽流；参数错误干净提示（不抛 traceback） |
| `agent_team/information_hub.py` | `run_roundtable` 提前返回的判据增加 `has_human_speaker`：**仅当存在人类发言者时**才在“无 ReAct 听众”的局（如离线人机）继续走发言轮，使人类能在白天讨论 / 警长竞选 / 狼人夜聊中发言。纯 Agent / 纯 demo 局行为不变 |

### 已知限制 / 取舍
- 离线 demo 机器人不参与 LLM 推理，其“发言”为占位文本；要体验真实对话请用真 LLM 对手。
- 错误提示目前为中文；`--players` 浮点值会被 `int()` 截断（属边角）。
- `--rules badge_flow`（选择配置文件）与 `--badge_flow`（开启警长机制）是两个独立开关，注意区分。

### 验证情况
经多 Agent 对抗式 review + 约 40 局实跑（含人类扮演者）覆盖：村民/狼人/预言家/女巫/守卫/猎人/丘比特各角色、
`--players` 6/9/11/12、`--human_seat`（含多座位与全 demo 配置）、`--badge_flow`、EOF 降级、边界报错；
并以真实 DeepSeek 对手确认人机对战可正常发言、无鉴权错误、无崩溃。默认全 Agent 路径回归通过、输出格式不变。
