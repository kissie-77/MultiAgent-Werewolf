# 人机混战信息隔离与交互修复

> **模块**：reports
> **状态**：active
> **最后更新**：2026-06-03
> **关联代码**：`src/llm_werewolf/interface/`、`src/llm_werewolf/ui/`、`src/llm_werewolf/game_runtime/`、`src/llm_werewolf/agent_team/`
> **关联测试**：`tests/interface/`、`tests/ui/`、`tests/game_runtime/`、`tests/agent_team/`
> **Agent Skill**：`.agents/skills/generated/interface/`、`.agents/skills/generated/ui/`、`.agents/skills/generated/game-runtime/`、`.agents/skills/generated/agent-team/`

本文记录 2026-06-02 至 2026-06-03 人机混战对局中暴露的信息隔离、日志展示、提示词污染与夜间决策元数据问题，以及本轮修复结果。

## 一、问题概述

### 1. 人类玩家视角日志不符合正常对局体验

人类玩家在白天发言前，会看到面向 LLM 的上下文块，例如：

- `【本轮已听到的发言】`
- `【子阶段·仅发言】`
- `【任务】白天公开讨论轮`
- “请先回应上一位发言者……”

这些内容对 LLM 有用，但人类玩家应通过正常滚动日志阅读其他玩家发言，轮到自己时只需要输入发言。

### 2. 白天发言与投票展示错位

人机混战中，`PLAYER_SPEECH` 在人类视角被缓冲到投票后才汇总展示，导致人类玩家还没看到后置位发言就进入投票；同时 `VOTE_CAST` 曾被设为投票者私密事件，使人类视角的投票表只显示自己可见的一票。

### 3. 夜间和分析信息泄露风险

人类村民视角不应看到狼人夜聊、狼人夜间旁白、信念矩阵、投票意向快照等上帝视角或私密信息。相关事件需要通过 `visible_to` 和 presenter 双层过滤。

### 4. Agent 白天发言 prompt 缺少本局角色池边界

基础 6 人局不存在守卫，但模型会凭通用狼人杀经验讨论“守卫守中”。角色池是本局稳定规则事实，应直接进入白天发言上下文；信念矩阵则保留在非人类 Agent 的工作记忆中，作为其内部推理辅助，而不是展示给人类玩家。

### 5. 女巫夜间行动元数据残留

女巫本次行动目标与事件中的 `decision` metadata 可能不一致。根因是事件日志读取 agent 上一次 `_last_decision_metadata`，结构化女巫决策无 target 时不会刷新该字段，导致旧目标残留。

### 6. 狼人首夜夜聊幻觉

首夜尚未发生白天发言，但狼人夜聊可能引用“某玩家白天活跃、带队”等未发生事实。需要在首夜狼队讨论上下文中明确禁止引用未来信息。

## 二、修复内容

### 1. 人类玩家输入界面降噪

- 发言类输入改为极简提示：
  - 白天公开发言：`请进行白天公开发言。`
  - 狼队夜聊：`请进行狼队夜聊发言。`
  - 警长竞选：`请进行警长竞选发言。`
  - PK 发言：`请进行 PK 发言。`
- 人类提示过滤 LLM schema、MsgHub、信念矩阵、工作记忆、任务块、公开发言边界等内部提示。
- 发言输入增加基本校验，拒绝过短、纯英文占位或重复字符。

### 2. 人类视角日志实时化

- 人类 viewer 下 `PLAYER_SPEECH` 立即打印，不再进入整轮发言缓冲。
- `VOTE_CAST` 改为公开事件，正式投票表可展示完整票型。
- `GAME_STARTED` 在 `setup_game` 前接入 `ConsolePresenter`，保证开局日志样式一致。
- 人机局开局显示人类玩家座位与身份。

### 3. 信息隔离收紧

- 狼人夜聊 `PLAYER_DISCUSSION` 和狼人夜间旁白仅狼队可见。
- `VOTE_INTENTION_SNAPSHOT`、`BELIEF_SNAPSHOT` 仅进入复盘/评测日志，不进入玩家 observation。
- 人类玩家发言上下文不追加 working memory，避免信念矩阵、狼队面板或内部分析进入人类输入界面。
- 非人类 Agent 的白天 working memory 注入保留信念矩阵、动态记忆和经验内容，用于局内内部推理。

### 4. Agent 白天发言上下文边界

- 白天公开发言 prompt 增加 `【本局角色池】`，列出本局实际存在的角色及数量。
- 增加 `【公开发言信息边界】`，提示 Agent 可在 `private_thought` 使用夜间私密信息，但 `public_speech` 不应无意识泄露刀口、验人、用药、守护等私密信息；明确跳身份除外。
- 非人类 Agent 的白天 working memory 注入保留信念类块；该内容只进入 Agent 决策上下文，不进入人类玩家输入界面。

### 5. 女巫行动元数据修复

- `WitchSaveAction`、`WitchPoisonAction` 创建时绑定本次 `_decision_metadata`。
- `ActionProcessorMixin._decision_data()` 优先读取 action 级 metadata，再回退 agent 缓存。
- 女巫夜间请求前清理旧 `_last_decision_metadata`，避免无行动或结构化无 target 时沿用旧值。

### 6. 狼队夜聊事实约束

- 首夜狼人讨论增加 grounding note，禁止引用未发生的白天发言、票型、活跃度、带队表现。
- 狼队夜聊提示改为综合前面已发言队友意见，不要求只回应上一位。

### 7. 2026-06-03 晚间追加修复

- `role_pool` 等本局固定事实从 `WorkingMemory` 的 `【稳定经验】` 中拆出，单独渲染为 `【本局固定信息】`，避免把当前局角色池误表达为长期经验。
- 最终狼刀事件 `WEREWOLF_KILLED` 改为对狼队与存活女巫可见：狼人知道最终刀口，女巫仍可按刀口决定是否用药；女巫救人、守卫守护等结果继续保持私密。
- 人类狼人提交刀人目标时，确认文案从“已提交目标”改为“已提交你的狼刀票；最终刀口以狼队结算为准”，避免把个人狼刀票误解成最终刀口。
- 人类狼人视角下，狼队夜聊 `PLAYER_DISCUSSION` 改为实时展示；全 Agent 上帝视角仍保留原分组面板，村民视角仍不可见狼人夜聊。

## 五、当前结论

本轮修复后，人机混战的核心边界为：

- 人类玩家看到的是正常对局日志和必要输入提示，不再暴露 LLM 内部 prompt。
- Agent 白天发言仍保留必要规则事实和信息边界，避免不存在角色污染与私密信息无意识外泄。
- 夜间私密频道、复盘分析事件、信念矩阵与投票意向快照不会进入普通玩家可见上下文；信念矩阵仅作为非人类 Agent 工作记忆的一部分进入其内部决策上下文。
- 最终狼刀对狼队与女巫可见，但救药、守护等抵消信息不向狼队泄露；人类狼人提交的是个人狼刀票，最终刀口仍由狼队票型与平票规则结算。
- 人类狼人夜聊改为实时滚动展示，后置狼人能在输入前看到前置队友发言。
- 女巫行动事件的目标 metadata 与本次真实行动一致。

后续若继续沉淀长期设计，应把稳定结论分别合入：

- `docs/interface/DESIGN.md`：人机混战 CLI / 输入体验
- `docs/ui/DESIGN.md`：ConsolePresenter viewer 过滤与实时展示
- `docs/game_runtime/DESIGN.md`：事件可见性规则
- `docs/strategy/DESIGN.md`：白天公开发言 prompt 边界
