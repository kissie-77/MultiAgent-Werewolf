## 3. 前端设计

### 3.1 双视角实时直播（观战 god / 人机 seat）

前端以 **SSE 下行 + POST 上行** 驱动，废弃旧 Express+Gemini mock，直连 FastAPI 并统一解 `ApiResponse{data}` 信封。

- **状态层** `store.ts`（Zustand）：`connectSpectate(runId)` 订阅 god 视角流，`connectSeat(runId,{seat,token})` 订阅 `view=seat&seat=&token=` 座位流；两者均把事件交给纯函数 `lib/gameReducer.ts` 折叠成可渲染 `GameState`。
- **可见性**：后端按 `Event.visible_to` 过滤，座位流仅含公开事件 + 本座位事件，**防作弊由后端天然保证**；前端无需也无法看到他人私密信息（狼聊、查验、信念矩阵）。
- **页面装配** `pages/GameApp.tsx`：依 `?run_id/view/seat/token` 决定 god / seat 订阅，挂载 3D 场景、`SpeechConsole`、`InsightDock`、人机输入面板与结算面板。

### 3.2 契约接缝：纯函数 mapper 层

后端页面端点与前端组件存在字段/形状差异，统一用「纯函数 mapper + 防御默认值」吸收（便于单测、零后端风险）：

| mapper | 作用 |
|--------|------|
| `lib/insightMap.ts` | 信念矩阵 / 投票意向（M2） |
| `lib/replayMap.ts` | 复盘页（snake→camel、概率 `×100`、座位 `→"P{n}"`、过滤 god 注入的 belief/vote 事件） |
| `lib/modelsMap.ts` | 模型榜：后端 `usage_stats`→页面 `models[]`、`win_rate` 0–1→百分比、`avg_mvp` null→0 |
| `lib/shareMap.ts` | 分享页：后端 snake_case `mvp_winner` + 小写阵营→前端形状、citation 兜底 |
| `utils/runRows.ts` | 战绩页：`villager`/`werewolf` 阵营→`GOOD`/`WEREWOLF`（注意 `"wolves"` 不含子串 `"wolf"`，需另判 `wolv`） |

### 3.3 M3 人机输入（座位真人决策）

引擎全程 `asyncio`；人类座位用 `WebHumanAgent`，轮到其决策时**挂起**并向 `HumanInputBroker` 注册待办、经广播推 `awaiting_input` 事件。前端：

- **唯一真人决策面**：`components/HumanInputPanel.tsx`（由 `store.pendingInput` 驱动的 portal 弹层），按 `kind` 渲染 座位选择 / 女巫三选一 / 是否 / 多选 / 发言框。
- **载荷归一化**：纯函数 `lib/humanInput.ts` 的 `buildHumanPayload` 把 UI 选择映射成后端 bridge 期望的文本（座位→`"3"`、救→`"救"`、毒→`"毒 [[N]]"`、是→`"1"`、发言→原文），经 `ApiClient.sendInput` POST `/api/v1/games/{id}/input`（带座位令牌）。
- **开局** `components/GameSetup.tsx`：人机模式选座位 + 警徽流，开局后用响应的 `player_token` 跳转座位视图。
- **注意**：遗留 `ControlPanel`/`SkillBar`/`CardDeck` 的技能/上警按钮指向已废弃的 `/api/game/*` mock，在 SSE 座位视图下为废件——真人决策**只**走 `HumanInputPanel`（后续应清理这些遗留控件）。

### 3.4 结算页：后端优先 + 崩溃兜底

真机测试发现旧 `GameOverPanel` 在座位链路（`gameState.players` 为空）会因 `calculateMVP()` 返回 `undefined`、随后解引用 `.role` 抛错而**整页白屏**，且 MVP 为前端重算（与后端不一致，曾在好人胜局把狼评为 MVP）。重构方案：

- 新增纯函数 `lib/gameOver.ts`（11 单测）：`resolveMvpView` / `resolveBoard` / `resolveWinnerIsGood`，对空名单**永不解引用**，并**优先采用后端 `mvp_scores` 排名**而非客户端重算。
- `GameOverPanel`：结算产物就绪后拉取 `getReplayData` 取真 MVP/底牌，渲染全程 null-safe，未就绪时显示「结算生成中」占位；新增 `userSeat` 用于「本人」标识。
- 新增 `components/ErrorBoundary.tsx` 包裹结算面板：单点渲染异常不再卸载整棵 React 树。

### 3.5 静态资源 / 角色立绘

角色立绘改为以站点根路径 `/material/*.png` 引用（不再从 `public/` 误 import 导致 Vite 返回 SPA 兜底 HTML）；`utils/roles.ts` 的角色→图映射同时支持后端英文角色名（`Seer`/`Werewolf`…）与中文展示名，并对狼系做兜底。

## 4. 后端改动（M3）

复用 M1 已建成的 SSE 直播半边（`EventBroadcaster` + `/games/{id}/stream` + 可见性过滤），新增人类输入半边：

| 组件 | 文件 | 说明 |
|------|------|------|
| `HumanInputBroker` | `interface/api/services/human_input.py` | 每 run 一个；`request()` 挂起 + 推 `awaiting_input(visible_to=[player_seat])`、`submit()` 幂等 resolve、超时安全兜底 |
| `WebHumanAgent` | `agent_team/agents/web_human_agent.py` | 复用 `HumanInteractiveAgent` 的 classify/归一化把决策 `await broker.request(...)`，bridge 文本路径零改动 |
| `create_agent` 分支 | `agent_team/agents/base.py` | `model=="web-human"` |
| `PlayerConfig` 内建模型 | `game_runtime/config/player_config.py` | `web-human` 免 base_url/key、不被 `model_env` 覆写 |
| 请求/响应扩展 | `interface/api/models/actions.py` | `StartGameRequest.human` + `StartGameResponse.player_token/stream_path` |
| 会话接线 | `interface/api/services/game_sessions.py` | 覆写座位为 `web-human`、生成座位令牌、把 broker 接进 agent |
| 输入路由 | `interface/api/routes/actions.py` | `POST /games/{id}/input`（令牌校验 403/404/409） |

**密钥安全**：原始 LLM Key 不进请求体（A7 原始 key 注入列为后续增强），AI 座位沿用服务端环境变量 key；座位令牌 `seat{N}-{run_id}` 仅防误操作、非鉴权。

## 7. 运行方式

```bash
# 后端（仓库根，注入有效 key，端口 8010）
DEEPSEEK_API_KEY=<key> OBS_READY_REQUIRE_LLM=0 uv run werewolf-api --port 8010
# 前端
cd frontend && VITE_API_PROXY=http://localhost:8010 npm run dev
```
浏览器开 `http://127.0.0.1:5173/`：「进入盘面」→ 观战或人机；人机座位视图 URL 形如 `/game?run_id=<id>&view=seat&seat=<N>&token=<player_token>`。
