## 2. 已修改问题

### 2.1 全 Agent 对战局点击后总是落到默认六人局

- 问题：前端启动对局时没有把真实 `player_count` 透传到启动请求，导致进入页面后统一按默认 6 人视图渲染。
- 修复：
  - `frontend/src/components/GameSetup.tsx`
  - `frontend/src/components/TimelinePlayback.tsx`
- 结果：全 Agent 对战现在会按真实对局人数进入和显示，不再统一回退成 6 人局。

### 2.2 主界面点击仪表盘白屏

- 问题：主页数据结构与前端页面读取字段不一致，旧字段映射失效后直接导致页面白屏。
- 修复：
  - `frontend/src/api/client.ts`
  - `frontend/src/api/types.ts`
  - `frontend/src/lib/homeMap.ts`
  - `frontend/src/lib/homeMap.test.ts`
- 结果：主页 `/api/v1/pages/home` 已通过 mapper 适配新版 `hero / stats_cards / recent_runs / quick_actions` 结构，仪表盘恢复正常跳转。

### 2.3 警长竞选后阶段切换不完整，警长动画长期占位

- 问题：
  - 前端没有正确识别 `sheriff_vote` phase；
  - 警长当选 overlay 的自动收起逻辑不稳。
- 修复：
  - `frontend/src/lib/gameReducer.ts`
  - `frontend/src/components/AlertOverlays.tsx`
- 结果：
  - 已补上 `sheriff_vote -> DAY_SHERIFF_VOTE` 映射；
  - 警长 overlay 改为独立 timeout 管理，避免重复触发后一直挂住页面。

### 2.4 复盘里的“邪灵恶狼幽影日志”没有正常显示

- 问题：前端复盘映射层把狼人阵营心智快照直接空返回，导致板块始终没有内容。
- 修复：
  - `frontend/src/lib/replayMap.ts`
  - `frontend/src/lib/replayMap.test.ts`
- 结果：现在会按 `wolf_camp_mind.jsonl` 的真实结构映射并显示该板块。

### 2.5 复盘时间线无法回拉到上面，只能看到最下面几条

- 问题：时间线容器高度与滚动逻辑有问题，导致滚动区只能停在底部附近。
- 修复：
  - `frontend/src/components/TimelinePlayback.tsx`
  - `frontend/src/pages/ReplayPage.tsx`
- 结果：时间线滚动恢复正常，可回拉查看上方历史事件。

### 2.6 模型 leaderboard 页面没有正常显示内容

- 问题：前端页面本身会渲染 `models`，但后端 `/api/v1/pages/models` 返回的 `usage_stats` 为空。
- 根因：模型统计聚合只尝试从 `run_context.roster` 读取 `ai_model`，漏掉了历史 run 里真实存在的 `launch_roster.json` 数据源。

### 2.7 排行榜板块没内容，漏读了一条数据源

- 修复：
  - `src/llm_werewolf/interface/api/services/runs.py`
- 具体改动：
  - 新增 `_load_launch_roster_models(run_dir)`；
  - 在 `aggregate_model_usage()` 中，当 `entry.ai_model` 取不到时，回退读取 `launch_roster.json` 里的 `players[].model`。
- 验证结果：
  - 本地聚合函数已能正确统计出 `deepseek-chat / kimi-k2.5 / deepseek-v4-flash`；
  - 重启 `werewolf-api` 后，模型排行榜即可读取现有历史 run 的统计结果。

### 2.8 “跳过等待”按钮点击后白屏

- 问题：这些按钮之前错误地伪造了固定夜间动作，而不是推进当前流程：
  - 狼人：默认 `NIGHT_KILL 2`
  - 预言家：默认 `NIGHT_INSPECT 4`
  - 女巫：默认 `NIGHT_SAVED_OR_POISON`
- 修复：
  - `frontend/src/components/ControlPanel.tsx`
- 结果：相关“跳过等待/跳过查验/跳过调药”按钮统一改为 `simulateNextAI`，语义恢复为“继续推进当前流程”，不再伪造角色决策。

### 2.10 run_id 观战无内容（已结束对局 / 无内存 session）

- 问题：后端重启后内存 session 丢失，`/game?run_id=...` 仅连上 SSE 快照，发言记录为空；可选对局未与服务端 `artifacts/runs/*/events.jsonl` 对齐。
- 修复：
  - **前端**：`store.connectSpectate` 先查 `GET /games/{id}/status`；非 live 时从 `GET /pages/replay` 读取时间线，`spectateLog.hydrateStateFromLogEvents` 回放；`gameReducer` 支持 `role_acting`、`data.speech`。
  - **前端**：开局「上帝观战」增加「载入历史对局」，列表来自 `GET /runs?replay_only=true`。
  - **后端**：SSE 快照在缺少 `god_roster.json` 时从 `events.jsonl` 重建 roster（`actions._roster_from_run_logs`）。
- 结果：已结束对局可脱离活跃 session，直接从磁盘日志观战；进行中对局仍走 SSE。

### 2.11 战绩 / 分享 / 首页「0 人局」

- 问题：部分 run 的 `player_count=0`、`winner_camp=null`（元数据未从日志推断）。
- 修复：
  - **后端**：`runs._scan_run_metadata` 按 `events.jsonl` → `god_roster` / `launch_roster` → `run_id` 中 `Np` 推断人数与胜方。
  - **前端**：`runRows` / `shareMap` / `homeMap` 在 API 返回 0 时用 `run_id` 兜底。
- 结果：战绩列表、分享卡、首页战报人数显示与可筛选逻辑一致。

### 2.12 供应商密钥设置（按供应商动态字段）

- 问题：设置弹窗写死 `ARK_*`，无法切换 DeepSeek / OpenAI 等。
- 修复：

  - **前端**：`GameSetup` 设置弹窗先选供应商再渲染 `env_name` 字段。
- 详见 [interface/PROVIDERS.md](../interface/PROVIDERS.md)。

## 3. 已发现但需协商问题

### 3.1 全局上帝视角按键没开时，聊天日志仍显示每个人身份

- 当前问题：日志区与上帝视角开关的显示边界不一致，可能造成“虽然没开按钮，但日志仍泄露身份”的体验问题。
- 待协商方向：是否需要把日志中的身份字段也纳入同一显示开关控制。

### 3.2 是否需要做“不同模型同台竞技”模式

- 当前问题：官网赛道任务里有这一项，但现有前端/入口没有明确提供此模式。
- 待协商方向：这是本阶段刚需，还是后续作为展示增强项单独做。

### 3.3 对局日志流程感不强，信息分散

- 当前问题：
  - 夜间狼人夜聊、白天发言、裁决等关键信息分散在聊天区、左上角、左下角；
  - 流程分界不够明确。
- 待协商方向：是否需要统一做更强的阶段标题、流程条或系统广播区。

### 3.4 右下角“查看神职”功能未完善，且与控制台角色介绍功能可能重复

- 当前问题：前端弹窗、控制台说明、角色说明页之间存在职责重叠。
- 待协商方向：
  - 保留两个入口但分工不同；
  - 或精简成一个主入口，避免重复维护。

### 3.5 “跳过等待”功能的交互语义还不够清晰

- 当前问题：
  - 从代码层面，它现在是“推进当前 AI 流程”；
  - 但从用户感知上，像是“跳过倒计时 / 自动替角色操作”，容易误解。
- 已完成修复：
  - 先修掉白屏；
  - 并确认它不再伪造角色动作。
- 待协商方向：按钮文案是否改为“继续推进 / 下一步 / 继续等待”更准确。
