# UI 设计

> **模块**：ui
> **状态**：active
> **最后更新**：2026-05-24
> **关联代码**：`src/llm_werewolf/ui/`

## 1. 目标

提供游戏的展示层：将游戏事件和状态以美观的方式展示给用户。支持控制台输出和终端 UI（TUI）两种展示方式。只读事件与状态，不修改游戏逻辑。

## 2. 范围

### 做

- 格式化游戏事件输出（Rich 库美化）
- 提供控制台展示器（ConsolePresenter）
- 提供终端 UI 应用（WerewolfTUI，基于 Textual）
- 定义 UI 组件（聊天面板、游戏面板、玩家面板）
- 管理样式和布局

### 不做

- 不修改游戏状态（归 `game_runtime`）
- 不执行 Agent 决策（归 `agent_team`）
- 不提供 Web 页面（归 `frontend`）
- 不装配游戏流程（归 `interface`）

## 3. 核心架构

```
GameEngine 产生 Event
    → ConsolePresenter 格式化输出
    → Rich Panel/Table 美化展示

GameEngine 产生 Event
    → WerewolfTUI 更新 UI 组件
    → Textual 框架渲染终端界面
```

## 4. 关键概念

| 概念 | 说明 |
|------|------|
| ConsolePresenter | 控制台展示器：使用 Rich 库格式化事件输出 |
| WerewolfTUI | 终端 UI 应用：基于 Textual 框架的交互式界面 |
| ChatPanel | 聊天面板：展示玩家发言记录 |
| GamePanel | 游戏面板：展示当前轮次、阶段、死亡信息 |
| PlayerPanel | 玩家面板：展示玩家列表、存活状态、座位号 |
| Styles | 样式定义：颜色主题、布局、组件样式 |

## 5. ConsolePresenter

### 5.1 职责

- 接收游戏事件，格式化输出
- 缓冲夜间行动，统一展示
- 处理特殊事件（警长、投票、死亡）
- 控制输出节奏，避免刷屏

### 5.2 事件处理

| 事件类型 | 处理方式 |
|----------|----------|
| 夜间行动 | 缓冲后统一展示 |
| 白天发言 | 实时展示 |
| 投票结果 | 表格展示 |
| 死亡信息 | 醒目提示 |
| 警长选举 | 专区展示 |

### 5.3 输出格式

使用 Rich 库提供：
- Panel：区块划分（夜晚行动、白天讨论、投票结果）
- Table：投票结果、玩家状态
- Text：带颜色的文本（角色名、座位号）

## 6. WerewolfTUI

### 6.1 架构

基于 Textual 框架的终端 UI 应用：

```
┌─────────────────────────────────────────┐
│ 游戏面板 (GamePanel)                     │
│ 轮次: 3 | 阶段: 白天讨论 | 存活: 6/9     │
├──────────────────┬──────────────────────┤
│ 玩家面板         │ 聊天面板             │
│ (PlayerPanel)    │ (ChatPanel)          │
│                  │                      │
│ 1号  存活  平民  │ 玩家1: 我觉得...     │
│ 2号  死亡  狼人  │ 玩家3: 我建议...     │
│ 3号  存活  守卫  │ 玩家4: 目前场上...   │
│ ...              │ ...                  │
└──────────────────┴──────────────────────┘
```

### 6.2 组件

| 组件 | 说明 |
|------|------|
| GamePanel | 顶部状态栏：轮次、阶段、存活人数 |
| PlayerPanel | 左侧玩家列表：座位号、存活状态、身份（死亡后显示） |
| ChatPanel | 右侧聊天区：实时滚动展示发言和事件 |

## 7. 样式系统

### 7.1 颜色主题

| 元素 | 颜色 |
|------|------|
| 狼人 | 红色 |
| 好人 | 绿色 |
| 神职 | 蓝色 |
| 死亡 | 灰色（删除线） |
| 系统消息 | 黄色 |

### 7.2 布局

- 控制台模式：单列输出，适合日志记录
- TUI 模式：三栏布局（状态栏 + 玩家列表 + 聊天区）

## 8. 接口与扩展点

| 入口 | 类型 | 说明 |
|------|------|------|
| `ConsolePresenter(locale)` | 构造函数 | 创建控制台展示器 |
| `presenter.present_event(event)` | 方法 | 展示单个事件 |
| `WerewolfTUI(...)` | 类 | Textual 终端 UI（见 `ui/tui_app.py`） |
| `run_tui(...)` | 函数 | TUI 入口包装 |

## 9. 依赖与边界

- `ui → game_runtime`（读事件与状态）
- `ui` 不修改 `GameState`
- `ui` 不依赖 `agent_team`、`evaluation`、`strategy`
- `interface` 可依赖 `ui`（TUI 入口）
