## 改了什么（按模块）

### 夜晚 / 狼队
- **梦魇狼顺序**：先跑前置阶段（含梦魇封锁），再狼队讨论 → 投票 → 结算 → 后置技能。
- **狼队夜聊**：去掉误挂的「白天放逐意向」采集，避免多余 LLM 调用和模型困惑。
- **狼票**：空票写日志；狼人投票校验「不能刀队友 / 不能刀自己」。

### 角色技能 / Action
- **白狼王**：阵营比较 + 角色名与 `RoleConfig.name` 对齐（`"White Wolf"`），技能可正常 `validate`。
- **隐狼 / 血月**：预言家查验显示阵营逻辑统一（隐狼显示好人）。
- **守墓狼 / 狼票 / 白天投票**：阵营与目标校验收紧。
- **行动处理**：`validate` 失败记 `action_rejected`；补梦魇狼、守墓狼、乌鸦、守墓人等事件日志；执行异常有日志。

### 警长选举 & 白天投票
- **平票**：第一次 PK 再投；第二次警徽流失 / 无人出局（警长、放逐各一套）。
- **异常**：参选、投票阶段 LLM 失败不再静默，有 error 日志。
- **文案**：硬编码中英文改为 `locale.get()`。
- **白天投票**：`asyncio.gather` 并发收票。

### 死亡 / 殉情 / 遗言
- 殉情按昼夜写入 `night_deaths` / `day_deaths`。
- 猎人开枪失败有 fallback + 日志；警徽移交、死亡技能提示走 locale。

### 工程与杂项
- 新增 **`core/roles/names.py`**：统一 `RoleNames` 与 `seer_apparent_camp()` 等，避免 `"WhiteWolf"` / `"Hidden Wolf"` 混用。
- 移除模块级重复 `dotenv.load_dotenv()`；prompt 解析小修（`re` 导入、`parse_yes_no`、多目标括号解析）。
- `game_config` 校验参数类型标注修正。
- **未做**：JSON Schema 约束 LLM 输出（试过，已回滚，仍用 `[[n]]` + AgentScope 结构化主路径）。

### roles 模块审查与修复（2026-05-24）
- **`__init__.py` 导出符号缺失**：补充 `VictoryGoal`、`ROLE_CATALOG`、`get_catalog`、`get_definition`、`RoleDefinition` 的导入，修复 `from llm_werewolf.core.roles import ROLE_CATALOG` 报错
- **`names.py` 阵营比较错误**：`player_camp_is` 中 `player.get_camp() == camp.value` 改为 `player.get_camp() == camp`（直接比较枚举）；`seer_apparent_camp` 中冗余 `Camp()` 包装改为条件判断
- **角色 description 硬编码英文**：将所有角色（狼人阵营 8 个 + 好人阵营 12 个 + 中立阵营 3 个）的 `description` 从英文改为中文，包括 `private_notes` 中的英文提示
- **`build_werewolf_team_context` 硬编码英文**：改为中文提示词

### types 模块审查与修复（2026-05-24）
- **`__init__.py` 导出符号缺失**：补充 `VictoryGoal` 的导入，修复 `from llm_werewolf.core.types import VictoryGoal` 报错
- **`Event.phase` 类型不一致**：`phase: str` 改为 `phase: GamePhase`，同步更新 `EventLogger.create_event` 签名
- **`PlayerProtocol.get_camp()` 返回类型错误**：从 `str` 改为 `Camp`，与实际实现一致
- **移除废弃的 `RoleProtocol.night_action`**：该方法已标记弃用且无调用方，从协议中移除
- **`ActionProtocol.execute()` 返回类型不精确**：从 `list[str]` 改为 `list[str] | None`，反映实际可能返回 `None` 的情况

### evaluation 模块审查与修复（2026-05-24）
- **`event.phase` 枚举序列化问题**：`checkers.py` 和 `runner.py` 中多处 `event.phase` 直接赋值到 data dict，改为 `event.phase.value` 确保 JSON 序列化一致
- **`InformationIsolationChecker._collect_sensitive_fragments` 重复提取**：移除重复的 `result = data.get("result")` 提取，因为 `_SENSITIVE_DATA_KEYS` 已包含 `"result"`
- **`runner.py` DemoAgent 导入路径错误**：从 `llm_werewolf.core.agent` 改为 `llm_werewolf.agents.base`
- **`EvaluationScenario.language` 默认值**：从 `"en-US"` 改为 `"zh-CN"`，与项目中文定位一致
- **`AsyncFlowChecker` 阶段比较**：`previous.phase` 和 `current.phase` 改为 `.value` 提取字符串后再比较

### 阵营 API 与存档扩展（2026-05-24）
- **`get_camp()` 统一为 `Camp` 枚举**：`Player.get_camp()` 返回 `role.camp`；`get_players_by_camp` / `count_alive_by_camp` 参数改为 `Camp`；UI / evaluation 调用方同步
- **`GameStateSnapshot` 扩展**：补充警长字段（`sheriff_id`、`sheriff_election_done`、`sheriff_votes`、`sheriff_tie_count`）及 `vote_tie_count`，读档时恢复警长玩家状态
- **发现问题文档**：记录盗贼/魔术师未实现（问题 5）、事件日志双轨可行性分析（问题 6）
