# 吕祎晗 - 修改记录

## 修复记录

### 修复 1：WhiteWolfKillAction 阵营比较错误
- **文件**：`src/llm_werewolf/core/actions/werewolf.py`
- **问题**：`validate()` 方法中使用 `self.target.get_camp() == "werewolf"` 字符串比较，但 `get_camp()` 返回 `Camp` 枚举，导致白狼击杀永远验证失败
- **修复**：导入 `Camp` 枚举，改为 `self.target.get_camp() == Camp.WEREWOLF`
- **状态**：✅ 已修复

### 修复 2：狼人讨论与 Nightmare Wolf 执行顺序错误
- **文件**：`src/llm_werewolf/core/engine/night_phase.py`
- **问题**：狼人讨论在 `run_pre_wolf_phase()` 之前执行，但 Nightmare Wolf 的封锁行动在 `run_pre_wolf_phase()` 中执行，导致梦魇狼封锁逻辑在讨论之后才生效
- **修复**：调整执行顺序为：前置阶段 → 狼人讨论 → 狼人投票 → 投票结算 → 后置阶段
- **状态**：✅ 已修复

### 修复 3：空投票场景缺少日志
- **文件**：`src/llm_werewolf/core/engine/night_phase.py`
- **问题**：`_resolve_werewolf_votes()` 在 `werewolf_votes` 为空时静默返回，没有记录事件，不利于调试
- **修复**：添加空投票日志记录，使用 `werewolf_no_votes` 本地化键
- **状态**：✅ 已修复

### 修复 4：警长竞选平票无处理
- **文件**：`src/llm_werewolf/core/engine/sheriff_election.py`、`src/llm_werewolf/core/game_state.py`
- **问题**：平票时只记录事件，不选出警长，但 `sheriff_election_done = True` 已设置，无法重新选举
- **修复**：
  - 添加 `sheriff_tie_count` 字段追踪平票次数
  - 第一次平票 → PK 发言 + 重新投票
  - 第二次平票 → 警徽流失，本局无警长
  - 新增 `_resolve_sheriff_result()`、`_handle_sheriff_tie()`、`_conduct_pk_speeches()` 方法
- **状态**：✅ 已修复

### 修复 5：候选人收集异常静默处理

```
夜晚开始
    ↓
┌─────────────────────────────────────┐
│ 前置阶段（Pre-Wolf）                  │
│ Cupid → Nightmare Wolf → Guard →    │
│ Guardian Wolf → Thief               │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 狼人讨论（Werewolf Discussion）       │

└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐

└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 投票结算（Resolve Votes）             │
│ 统计票数 → 确定 werewolf_target      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 后置阶段（Post-Wolf）                 │

└─────────────────────────────────────┘
    ↓
死亡结算（resolve_deaths）
    ↓
狼人闭眼 → 夜晚结束
```
