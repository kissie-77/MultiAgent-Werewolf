# 工程架构重构计划

## 目标

本计划用于把当前项目从 `core / adapter / integration / agents / evaluation / ui` 的目录分层，逐步收口为更清晰的工程职责模块：

| 目标模块 | 职责 | 当前主要对应 |
| --- | --- | --- |
| `game + runtime` | 狼人杀规则、角色、动作、状态、流程调度 | `core/` |
| `agent_team` | AgentScope 多智能体运行、Agent 创建、消息隔离、LLM 调用封装 | `agent_team/` |
| `strategy` | Prompt、角色策略、结构化决策、投票意向 | `strategy/role_prompts.py`、`core/prompts/`、`core/decisions.py` |
| `interface` | 启动入口、模式选择、CLI/TUI/评估命令 | `interface/` |
| `ui` | 对局界面、玩家面板、聊天面板、人机交互组件 | `ui/` |
| `evaluation` | 评估、复盘、bad case、自进化闭环 | `evaluation/`、`docs/prompt_tuning.md` |

## 当前优先决策

### AgentScope 主线

- 最终运行主线统一为 `src/llm_werewolf/agent_team/agentscope_agent.py`。
- AgentScope 玩家主实现已迁入 `src/llm_werewolf/agent_team/agentscope_agent.py`。
- `src/llm_werewolf/integration/agentscope.py` 与 `src/llm_werewolf/adapter/agent.py` 不再维护独立旧版 `AgentScopeWerewolfAgent` 实现，只作为兼容导出入口。
- `src/llm_werewolf/agent_team/serial_calls.py` 已转为正式默认模块，用于保护 AgentScope 调用的串行限流；`adapter/serial_calls.py` 仅兼容导出。
- 旧 OpenAI/`LLMAgent` 单轮接口不再作为正式主路径，暂时保留为 legacy/baseline，后续框架迁移时删除或隔离。

### Prompt 主线

- 新的角色策略内容以 `src/llm_werewolf/strategy/role_prompts.py` 中的 `RolePrompts` 为准。
- `src/llm_werewolf/adapter/prompts.py` 仅保留兼容导出，避免旧 import 失效。
- `src/llm_werewolf/core/prompts/` 作为未来统一 `PromptManager` 入口。
- 后续迁移方向是把 `adapter/prompts.py` 的角色策略卡接入或迁入 `core/prompts/PromptManager`，让 AgentScope 主线统一从 PromptManager 构建 prompt。

### core 与 adapter 边界

- 本阶段不重写核心调用链。
- 长期目标是让 `core` 只表达游戏规则和“需要一次玩家决策”，通过 `PhaseInteraction` / `InformationHub` / `WerewolfAdapterBridge` 隔离 AgentScope、OpenAI 或人类输入。
