# 狼人杀对战测试 · 技能使用验证报告

> 范围：人机对战 + 纯 Agent 对战；后端模型 **doubao-seed-2.0-pro** 与 **deepseek-v4-flash**。
> 重点：人类玩家与 Agent 的 **技能使用 / 投票 / 发言 / 技能对象** 能否被系统正确识别、并被其他 Agent 获知。
> 测试日期：2026-05-26 ｜ 测试者以「人类玩家」身份亲自完整对战两局。

---

## 一、总结论

**PASS** — 在 doubao 与 deepseek 两个后端下，人类玩家与纯 Agent 的技能、投票、发言、技能对象均能被系统正确识别；人类公开发言能广播进所有 LLM Agent 的记忆，并被逐字理解、触发分阵营策略反应。

我亲自打的两局都获胜：

| 对局 | 我的角色 | 对手 | 结果 |
|---|---|---|---|
| 人 vs deepseek | 预言家 Seer | 5 × deepseek-v4-flash | **好人胜**（放逐两只狼） |
| 人 vs doubao | 女巫 Witch | 5 × doubao-seed-2.0-pro | **好人胜**（救人 + 毒杀收尾） |

发现 **1 个重要可靠性问题**（deepseek 静默丢弃决策）与 **1 个性能问题**（doubao 极度啰嗦导致缓慢），详见第五节。

## 三、人类玩家 — 四项检查（核心）

### 1. 技能 + 技能对象 — ✅ 识别正确

**预言家（deepseek 局）**
```
🔮 预言家查验了 Player2
（私密返回）第1夜：查验Player2，结果为狼人
🔮 预言家查验了 Player3
（私密返回）第2夜：查验Player3，结果为狼人
```
两晚查验目标与真实阵营结果均正确返回。

**女巫（doubao 局）**
```
(收到输入 seq=1: '救' -> normalized='救')      → 💊 女巫救了 玩家4   → 平安夜（玩家4 未死）
(收到输入 seq=4: '毒 2' -> normalized='毒 [[2]]') → ☠️🧪 女巫对 玩家2 使用毒药 → ☠️ 玩家2 被女巫毒杀
```
- 解药用完后，系统正确显示 `解药可用：否`（药剂状态正确跟踪）。
- 输入归一化正确：`救`→save、`毒 2`→`毒[[2]]`→poison seat2、座位号→target；非法输入就地重试。

### 2. 投票 — ✅ 识别并正确计票

```
deepseek： 🗳️ Player1 投票给 Player2 ｜ Player2: 2.0 票 - Player1, Player5
           （平票进 PK，我再投出第二只狼 → Player3 被投票淘汰，身份是 Werewolf）
doubao：   🗳️ 玩家1 投票给 玩家3 ｜ 玩家3: 4.0 票 - 玩家1, 玩家4, 玩家5, 玩家6
           （玩家3 被投票淘汰，身份是 Werewolf）
```
人类票被正确识别、归属到本人、计入统计，并参与平票 PK。

### 3. 发言被系统识别 + 被其他 Agent 获知 — ✅（最关键，证据确凿）

**deepseek 局**：我（预言家）起跳查杀 Player2 后，狼人 Player2 的私念逐字复述了我的发言：
> Player2(thinking)：**我们收到了 Player1 的发言，他说他是真预言家，昨晚查验了 Player2 是狼人**，他要求大家投 Player2 出局。现在我是 Player2，被悍跳预言家查杀了，我必须立即反击……

随后 Player2 悍跳预言家反咬，狼同伴 Player3 私念 *“为保护狼队友，我坚持投1号”*，而村民 Player5 据我的发言抓住对方矛盾、跟我一起投狼。

**doubao 局**：我（女巫）公开“救了4号”后：
> 玩家3（狼）：**搞笑了啊1号，你哪来的脸跳女巫？我才是真女巫，昨天夜里我开解药救的就是4号**……（镜像我的发言悍跳女巫）
> 玩家4（真预言家）：**感谢1号女巫的银水佐证我的身份**，3号是我昨晚验出来的铁狼。

→ 人类公开发言经 MsgHub 广播进**所有** LLM Agent 的记忆，被**逐字理解**并触发分阵营的策略反应；其他玩家会跟随或对抗。

## 五、发现与建议

### ⚠️ 发现 1：deepseek-v4-flash 结构化输出可靠性问题（重要）

**现象**（pure_ds 纯 deepseek 局）：
- 女巫选择了 `save` 但目标 Player3 仍死亡，**全局 0 个 `💊` 事件**；
- 白天正式投票 5 人**仅 2 票生效**，多个 Agent 自述“我投了 2 号”却没被计入；
- Agent 自身困惑：*“我选了 save 但人没救”*、*“我投了 2 号但没计票”*。

**根因**：deepseek-v4-flash 是重推理模型，常在 `max_tokens=2048`（`src/llm_werewolf/agent_team/factory.py:create_react_agent`）内耗尽 token 仍未发出 `generate_response` 工具调用；`src/llm_werewolf/agent_team/bridge.py` 的 `request_witch_night_choice` / `request_seat_choice` 以 `except Exception: pass` **静默回落**到默认值（女巫 none / 投票弃票），把模型的格式失败悄无声息地转成了真实的“决策丢失”，且没有任何日志。

**对比**：doubao 走同一代码路径，救人/投票全部正常；deepseek 的**发言**与**预言家查验**（输出更短）也正常 → 属于「长输出 + 静默回落」组合下的偶发问题。

**建议**：
1. 夜间/投票决策提高 `max_tokens`（或针对决策类调用单独放宽）；
2. 回落时 `log.warning` 而非 `pass`，至少暴露“决策被丢弃”；
3. 增加结构化重试；或评估 `deepseek-v4-pro` / 非推理模型。

### ⚠️ 发现 2：doubao-seed-2.0-pro 极度啰嗦 → 速度很慢

每个决策产出海量 reasoning，并叠加额外的「Now generate a text response」ReAct 文本回合；单是 day-1 讨论就跑了约 20 分钟 / 3000+ 行。`vote-intention` 在每条发言后对全体采集，进一步放大开销。

**建议**：关闭或抽样 vote-intention 采集；限制 ReAct 文本回合数；或对决策类调用使用更短的提示。

### 发现 3（轻微）：女巫夜 2 提示自相矛盾

提示同时出现 `今晚狼人目标是玩家4` 与 `解药已耗尽，本夜不会告知刀口身份` —— 解药用完后仍透露了刀口（疑似观测上下文未做信息隔离）。影响小，建议核查 `_merge_private_context` / 观测构建对女巫的字段。

## 六、人类玩家体验反馈

- **提示清晰**：每次轮到我，系统给出精简提示 + 输入格式（`救/毒 座位号/0`、座位号、≥15 字发言），面向 LLM 的 schema 噪声已被过滤。
- **负担合理**：人类被排除在 vote-intention 采集之外（因无 `agentscope_agent`，被 MsgHub audience 过滤），一局只需「夜间技能 + 每日发言 + 每日投票」各一次。
- **发言真能左右牌局**：两局都靠“起跳神职 + 带队投票”赢下；doubao 局里我（真女巫）甚至被狼人“悍跳女巫”对刚，最终靠预言家银水背书 + 毒杀收尾。整体体验接近真人局。

## 七、复现方式

```bash
# 人 vs deepseek（强制人类为预言家）
WW_CONFIG=configs/llm-6p-deepseek.yaml WW_HUMAN_SEAT=1 WW_FORCE_ROLE=Seer \
WW_IO_DIR=_play/human_ds PYTHONUTF8=1 PYTHONUNBUFFERED=1 \
.venv/Scripts/python.exe -u _play/ww_play_driver.py > _play/human_ds/console.txt 2>&1

# 人 vs doubao（强制人类为女巫）
WW_CONFIG=configs/llm-6p-doubao.yaml WW_HUMAN_SEAT=1 WW_FORCE_ROLE=Witch \
WW_IO_DIR=_play/human_doubao ... 同上

# 纯 Agent 局：WW_HUMAN_SEAT=0（无人类，跳过赛后评估）
```
人类输入：座位号；女巫 `救` / `毒 座位号` / `0`；发言为完整中文（≥15 字）。
应答方式：读 `$WW_IO_DIR/pending.json` 看待输入，写 `answer.json`（`{"seq":N,"text":"..."}`）。
