# 真实对局测试清单（PostGame / MVP）

> 在跑真 API 对局前确认；对局结束后用本清单验收 `runs/<id>/` 产物。

## 1. 跑局前

- [ ] `config` 中玩家 `model` 非 `demo`/`human`，且 `api_key_env` 已在环境变量中配置
- [ ] `PlayersConfig.prompt_version` 为 `v2`（或与目标 Prompt 一致）
- [ ] 投票意向追踪已开启（对局内会产生 `vote_intention_snapshot` 事件）
- [ ] 明确 `run_dir` 路径（CLI / `finalize_run` 会写入该目录）

## 2. 对局结束自动触发

- `interface/finalize_run` → `run_post_game_pipeline`（**完整**流水线，含 LLM 复盘，除非手动 `skip_llm`）
- 批量正确性 `correctness/runner`：只要有 `events.jsonl` 即跑 PostGame（`skip_llm=True`）

## 3. 必查产物（`runs/<timestamp>/`）

| 文件 | 说明 |
| --- | --- |
| `events.jsonl` | 全量事件 |
| `vote_intentions.jsonl` | 有公开/狼队讨论时应有记录 |
| `mvp_scores.json` | `schema: mvp_scores_v2`，含 `benefit` 维度与金句 |
| `benefit_scores.json` | `schema: benefit_scores_v2`，`phase: v1_complete` |
| `views/score_contexts/*.md` | 分维度上下文（复盘 LLM 唯一材料） |
| `game_quality_report.md` | 人读总览 |
| `post_game_steps.json` | 每步 ok/failed/skipped |
| `prompt_proposals.json` | 含 `mvp_golden_speech` / `llm_suggestion` |
| `role_skills.json` | `schema: role_skills_v2`，含背景/引用 |

## 4. 失败时

1. 打开 `post_game_steps.json`，看 `failed` 步骤与 `error`
2. 若 `llm_replay` 为 `skipped`：检查 API Key、`evaluation` 配置段、或先用 `skip_llm=True` 验证规则层
3. 若 `mvp_scores` 置信度 `low`：多为缺少 `vote_intentions.jsonl`

## 5. 仅重跑 PostGame（不重开对局）

```bash
cd MultiAgent-Werewolf
uv run python -c "
from pathlib import Path
from llm_werewolf.evaluation.post_game.pipeline import run_post_game_pipeline_sync
run_post_game_pipeline_sync(Path('runs/<你的run目录>'), skip_llm=False)
"
```

## 6. Phase 2 仍未要求（真测可忽略）

- Skill 按 benefit 阈值筛选
- AgentScope LLM 自动写 Skill 正文（当前为规则门控 + 模板卡片）
